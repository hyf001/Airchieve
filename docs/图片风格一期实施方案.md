# 图片风格一期实施方案：基础数据与管理接口

## 1. 阶段目标

第一期只建设图片风格的基础能力，不接入绘本生成主链路。

本阶段完成后，系统应具备：

- 图片风格独立数据模型
- 图片风格版本数据模型
- 图片风格参考图独立数据模型
- 参考图与风格版本一对多关联
- 管理员可创建、编辑、启用、停用图片风格
- 管理员可创建版本草稿、发布版本
- 普通用户可查询已启用且有当前版本的画风列表
- `Storybook` 表具备后续绑定画风的字段

本阶段不做：

- 绘本生成时使用画风
- 分镜生成时使用画风摘要
- 页面图片生成 prompt 改造
- 自动解析画风
- 管理员试生成
- 小程序改造

## 3. 数据模型

### 3.1 ImageStyle

新增表：

```text
image_styles
```

建议字段：

```text
id
name
description
cover_image
tags
current_version_id
is_active
sort_order
creator
modifier
created_at
updated_at
```

字段说明：

- `tags` 建议使用 JSON 字段保存字符串数组。
- `current_version_id` 指向当前生效版本。
- `is_active=false` 时，普通用户不可选择该画风。
- `sort_order` 用于前端展示排序。

### 3.2 ImageStyleVersion

新增表：

```text
image_style_versions
```

建议字段：

```text
id
image_style_id
version_no
style_summary
style_description
generation_prompt
negative_prompt
status
creator
created_at
published_at
```

字段说明：

- `status` 可取 `draft`、`published`。
- 已发布版本不直接修改，如需调整则复制为新草稿。

### 3.3 ImageStyleReferenceImage

新增表：

```text
image_style_reference_images
```

建议字段：

```text
id
image_style_version_id
url
is_cover
sort_order
note
creator
created_at
updated_at
```

字段说明：

- `image_style_version_id` 指向所属风格版本。
- 每个版本可以关联多张参考图。
- 参考图单独建表，便于后续新增、修改、删除、设置封面图和调整排序。
- `is_cover=true` 的图片可作为该版本主参考图。

参考图请求结构：

```json
[
  {
    "url": "https://...",
    "is_cover": true,
    "sort_order": 0,
    "note": "主参考图"
  }
]
```

### 3.4 Storybook 字段预留

修改 `app/models/storybook.py`，新增：

```text
image_style_id
image_style_version_id
```

说明：

- 第一期只加字段，不接入生成逻辑。
- 历史数据不迁移。
- `template_id` 暂时保留，避免影响旧数据读取。

## 4. 后端接口

新增路由文件：

```text
app/api/v1/image_style_api.py
```

并在：

```text
app/api/v1/router.py
```

注册：

```text
/image-styles
```

### 4.1 获取可用画风列表

```http
GET /api/v1/image-styles?is_active=true&limit=100&offset=0
```

普通用户只返回：

- `is_active=true`
- 存在 `current_version_id`
- 当前版本状态为 `published`

响应字段：

```json
{
  "id": 1,
  "name": "柔和水彩绘本风",
  "description": "低饱和、柔和、水彩纸纹、温暖光线",
  "cover_image": "https://...",
  "tags": ["水彩", "温暖"],
  "current_version_id": 11,
  "current_version_no": "v1",
  "sort_order": 10
}
```

### 4.2 获取画风详情

```http
GET /api/v1/image-styles/{style_id}
```

普通用户访问停用画风时返回 `404` 或 `403`。

### 4.3 创建画风

```http
POST /api/v1/image-styles
```

仅管理员可用。

请求字段：

```json
{
  "name": "柔和水彩绘本风",
  "description": "适合温暖治愈类绘本",
  "cover_image": "https://...",
  "tags": ["水彩", "温暖"],
  "is_active": true,
  "sort_order": 10
}
```

### 4.4 更新画风基础信息

```http
PUT /api/v1/image-styles/{style_id}
```

仅管理员可用。

可更新：

- 名称
- 描述
- 封面图
- 标签
- 启用状态
- 排序

### 4.5 创建画风版本草稿

```http
POST /api/v1/image-styles/{style_id}/versions
```

仅管理员可用。

请求字段：

```json
{
  "style_summary": "整体温暖、低饱和、儿童绘本式画面。",
  "style_description": "柔和的儿童绘本插画风格...",
  "generation_prompt": "参考该画风的整体视觉特征...",
  "negative_prompt": "不要生成写实摄影风、3D游戏风、水印、乱码文字...",
  "reference_images": [
    {
      "url": "https://...",
      "is_cover": true,
      "sort_order": 0,
      "note": "主参考图"
    }
  ]
}
```

说明：

- `reference_images` 仅用于创建版本草稿时批量初始化参考图。
- 参考图实际保存到 `image_style_reference_images` 表。
- 草稿创建后，参考图的新增、修改、删除通过独立参考图接口完成。

版本号规则：

- 如果是该画风第一个版本，生成 `v1`。
- 否则按已存在版本数量递增生成 `v2`、`v3`。

### 4.6 获取版本列表

```http
GET /api/v1/image-styles/{style_id}/versions
```

仅管理员可用。

用于管理端查看版本历史。

### 4.7 发布版本

```http
POST /api/v1/image-styles/{style_id}/versions/{version_id}/publish
```

发布时校验：

- 版本属于该画风
- `style_summary` 不为空
- `style_description` 不为空
- `generation_prompt` 不为空
- `negative_prompt` 不为空
- 该版本下的参考图至少 1 张

发布成功后：

- 版本状态改为 `published`
- `ImageStyle.current_version_id` 更新为该版本

### 4.8 新增版本参考图

```http
POST /api/v1/image-styles/{style_id}/versions/{version_id}/reference-images
```

仅管理员可用。

请求字段：

```json
{
  "url": "https://...",
  "is_cover": false,
  "sort_order": 1,
  "note": "辅助参考图"
}
```

### 4.9 更新版本参考图

```http
PUT /api/v1/image-styles/{style_id}/versions/{version_id}/reference-images/{image_id}
```

仅管理员可用。

可更新：

- 图片地址
- 是否封面图
- 排序
- 图片备注

### 4.10 删除版本参考图

```http
DELETE /api/v1/image-styles/{style_id}/versions/{version_id}/reference-images/{image_id}
```

仅管理员可用。

## 7. 异常处理

### 7.1 画风不存在

返回：

```text
404 画风不存在
```

### 7.2 非管理员访问管理接口

返回：

```text
403 无权限操作图片风格
```

### 7.3 发布配置不完整

返回：

```text
422 画风配置不完整
```

### 7.4 参考图为空

创建版本草稿允许先为空还是禁止，需要二选一。

推荐：

- 草稿允许为空
- 发布时必须至少 1 张

### 7.5 参考图不存在

返回：

```text
404 画风参考图不存在
```

### 7.6 删除策略

第一期不建议做物理删除。

推荐：

- 通过 `is_active=false` 停用画风
- 版本不删除

## 8. 验收标准

第一期完成后，应满足：

1. 后端存在 `ImageStyle`、`ImageStyleVersion`、`ImageStyleReferenceImage` 模型。
2. `Storybook` 已预留 `image_style_id`、`image_style_version_id`。
3. 管理员可以创建图片风格。
4. 管理员可以创建风格版本草稿。
5. 管理员可以发布版本。
6. 普通用户只能查询已启用且有已发布版本的画风。
7. 参考图列表保存在独立表 `image_style_reference_images` 中。
8. 不影响现有 `template_id` 绘本生成流程。

## 9. 建议测试

后端：

- 创建画风成功
- 更新画风成功
- 创建版本草稿成功
- 发布完整版本成功
- 发布不完整版本失败
- 普通用户不能访问管理接口
- 停用画风不出现在用户列表

前端：

- `imageStyleService.ts` API 请求路径正确
- 401 时触发登录失效逻辑
- 写操作错误能展示后端错误信息

## 10. 不做事项

第一期明确不做：

- 自动解析画风
- 管理员试生成
- 分镜生成接入画风
- 页面生图接入画风参考图
- Web 创建绘本页面切换到画风
- 小程序改造
