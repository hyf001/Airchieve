# 绘本视觉锚点与上下文收敛第二阶段实施方案

## 1. 阶段目标

第二阶段不再把视觉锚点设计成“强视觉真相”，而是把它降级为：

```text
生成前的轻量连续性提示
```

它只解决一个小问题：

```text
在没有可靠图片连续性之前，给重复角色或关键物一个短设定，减少随机漂移。
```

本阶段不做：

- 不做生成后 QA。
- 不做自动修复。
- 不从参考图反推锚点。
- 不做锚点编辑后台。
- 不单独建锚点表。
- 不做场景锚点。
- 不把锚点作为高于上一页图片的硬约束。

## 2. 核心结论

视觉锚点仍然有必要，但只能保守使用。

优先级：

```text
1. 上一页图片连续性
2. 用户上传角色参考图
3. 视觉锚点
4. 锁定画风
5. 当前页分镜
```

也就是说：

- 如果上一页已经生成了红杯子，即使文本锚点写的是蓝杯子，后续也应延续上一页红杯子。
- 如果用户上传了角色参考图，角色外观以参考图和上一页图片为准，不生成角色锚点。
- 锚点只用于提醒模型“这里有一个需要稳定的对象”，不是最终裁判。

## 3. 锚点范围

### 3.1 保留的锚点

本阶段只保留两类锚点：

| 类型 | 使用条件 | 示例 |
|---|---|---|
| `character` | 仅当没有上传角色参考图时生成 | 小熊、妈妈、小女孩 |
| `object` | 多页出现或剧情关键物 | 红杯子、钥匙、书包 |

不做 `scene` 锚点。场景连续性优先交给上一页图片和当前页分镜。

### 3.2 数量限制

| 类型 | 限制 |
|---|---|
| 主角 / 角色锚点 | 最多 3 个 |
| 关键物锚点 | 最多 5 个 |
| 每页注入锚点 | 最多 3 个 |

锚点越少越好。普通背景物、一次性道具、装饰物不进入锚点。

### 3.3 颜色限制

颜色锚点只能做粗粒度提示，不能做精确色值控制。

可以写：

```text
红色杯子
蓝色围巾
黄色书包
```

不要写：

```text
偏橘大红
低饱和粉红
草莓红
暖调樱桃红
```

如果已经有上一页图片，颜色以上一页图片为准。

## 4. 角色参考图规则

上传图片入口只保留一种产品语义：

```text
角色参考图
```

工程侧只判断“角色参考图入口是否有上传图片”，不判断图片内容到底是不是角色。

| 场景 | 角色锚点策略 |
|---|---|
| 有角色参考图上传 | 不生成 `character` 锚点 |
| 无角色参考图上传 | 可以生成少量 `character` 锚点 |

系统首版不做图片内容识别，只做格式、大小、数量校验。UI 需要明确提示：

```text
请上传主角或主要角色参考图，不要上传物品图、场景图或风格图。
```

如果用户误把物品或场景上传到角色参考图入口，系统仍按角色参考图处理。这属于用户输入语义错误，首版不拦截。

## 5. 输入侧交互

创建绘本的输入必须在第一步一次性收齐。后续页面只做确认和修改，不再收集新内容。

第一步收集：

- 故事主题 / 正文 / 创作指令。
- 页数、年龄段、语言。
- 图片风格。
- 角色参考图。
- 图片比例、尺寸、生成模型。

流程：

```text
输入页
-> 生成故事 / 分镜 / 轻量锚点
-> 分镜确认
-> 创建绘本并生成图片
-> 完成
```

确认阶段允许：

- 修改故事文本。
- 修改页面分镜。
- 回到输入页替换角色参考图。
- 重新生成故事 / 分镜 / 锚点。

确认阶段不允许：

- 新增角色参考图。
- 新增角色、物品、场景设定。
- 临时上传物品图或场景图作为参考。

原因：

```text
锚点生成必须知道是否存在角色参考图。
角色参考图必须早于锚点生成。
has_character_reference_images 必须是稳定输入。
```

## 6. 分镜结构

第二阶段新生成和重生成分镜只使用弱结构字段，不再产出：

```text
scene
characters
shot
color
lighting
```

每页分镜结构：

| 字段 | 说明 |
|---|---|
| `summary` | 当前页视觉摘要 |
| `visual_brief` | 自然语言画面目标 |
| `anchor_refs` | 本页引用的锚点 ID |
| `must_include` | 本页必须出现的少量元素 |
| `composition` | 轻量构图要求 |
| `avoid` | 本页禁止项 |

图片 prompt 只使用这些字段。

## 7. 锚点生成方式

首版建议把锚点合并到分镜生成里，不额外增加一次模型调用。

分镜生成返回：

| 顶层字段 | 说明 |
|---|---|
| `anchors` | 绘本级轻量锚点 |
| `pages` | 每页故事文本和分镜 |

生成规则：

- 如果有角色参考图，不生成 `character` 锚点。
- 如果没有角色参考图，可以生成最多 3 个 `character` 锚点。
- 只生成多页出现或剧情关键的 `object` 锚点。
- 不生成 `scene` 锚点。
- 每页 `anchor_refs` 只能引用已存在的锚点。
- 每页最多引用 3 个锚点。

如果模型返回了不符合规则的锚点，后端 normalize 时直接过滤。

## 8. 数据存储

首版不单独建锚点表。

只增加一个轻量字段：

| 位置 | 内容 |
|---|---|
| `Storybook.visual_anchors` | 绘本级锚点 JSON |
| `Page.storyboard.anchor_refs` | 页面引用的锚点 ID |

图片生成前：

```text
读取 Page.storyboard.anchor_refs
-> 从 Storybook.visual_anchors 中取出本页锚点
-> 注入图片 prompt
```

## 9. 图片 Prompt 改造

图片生成 prompt 增加两块：

```text
本页视觉锚点
锚点边界规则
```

锚点边界规则必须表达：

- 上一页图片优先于文本锚点。
- 角色参考图优先于文本锚点。
- 锚点只约束少量故事关键属性。
- 颜色锚点只做粗粒度提示。
- 如果锚点和上一页图片冲突，以上一页图片为准。

不要把整本锚点表注入 prompt，只注入本页 `anchor_refs` 对应的锚点。

## 10. 失败降级

锚点是增强能力，不是主流程硬依赖。

以下情况直接降级为无锚点：

- 锚点为空。
- 锚点 ID 非法。
- 返回了 `scene` 锚点。
- 有角色参考图时返回了 `character` 锚点。
- 页面 `anchor_refs` 引用了不存在的锚点。
- `Storybook.visual_anchors` 保存失败。

降级后：

- 保留故事和分镜。
- 清理非法 `anchor_refs`。
- 图片生成不注入锚点。
- 不重跑整本分镜。

## 11. 实施步骤

1. 调整输入页，一次性收齐故事、风格、角色参考图和基础参数。
2. 将上传图片入口收敛为“角色参考图”，只做格式、大小、数量校验。
3. 给 `Storybook` 增加 `visual_anchors` JSON 字段。
4. 更新 `Storyboard` 类型，使用 `summary / visual_brief / anchor_refs / must_include / composition / avoid`。
5. 改造 Gemini / 豆包分镜生成，返回 `anchors + pages`。
6. 增加锚点 normalize：过滤非法锚点、`scene` 锚点、超量锚点，以及有角色参考图时的 `character` 锚点。
7. 图片生成前根据 `anchor_refs` 组装本页锚点。
8. 修改 Gemini / 豆包图片 prompt，加入锚点边界规则。
9. 增加必要日志和异常处理。
10. 用短绘本手工验证效果。

## 12. 验收标准

功能验收：

- 输入页一次性收齐所有生成信息。
- 分镜确认页不提供新增参考图或新增设定入口。
- 有角色参考图时，不生成 `character` 锚点。
- 无角色参考图时，可以生成少量 `character` 锚点。
- 不生成 `scene` 锚点。
- 新分镜不包含 `scene / characters / shot / color / lighting`。
- 锚点保存到 `Storybook.visual_anchors`。
- 页面引用保存到 `Page.storyboard.anchor_refs`。
- 图片 prompt 只注入本页相关锚点。
- 锚点异常时不阻塞绘本生成。

质量验收：

- 关键物是否比第一阶段更稳定。
- 没有角色参考图时，主角外观是否更稳定。
- 有角色参考图时，角色外观是否仍由参考图主导。
- 锚点没有覆盖上一页已经生成的视觉事实。
- 没有因为锚点造成颜色、形状等明显冲突。
