# 绘本视觉锚点与上下文收敛方案

## 1. 背景

当前绘本生成已经支持选择图片风格，并在图片生成阶段传入：

- 故事全文上下文
- 当前页故事文本
- 当前页视觉摘要
- 当前页分镜
- 上一页图片
- 角色参考图
- 风格版本描述
- 风格主图片

这套方式能让模型理解完整故事，但也带来两个关键问题：

1. 完整故事上下文过长，容易抢走模型注意力。模型可能为了“理解整本书”而忽略当前页的硬性视觉要求。
2. 连续性要求过于宽泛。比如“保持前后页一致”无法稳定约束杯子颜色、衣服颜色、角色配饰等细节。

实际表现可能是：

- 用户选择了风格，但成品风格不像。
- 前一页是红色杯子，后一页变成绿色杯子。
- 第一页的错误风格被上一页参考继续放大。
- 传入越多上下文，模型越容易在多个目标之间摇摆。

本方案目标不是继续增加 prompt 信息量，而是反过来做上下文收敛：只给模型当前页真正需要的信息，并把少数关键视觉元素变成强约束。

## 2. 核心原则

### 2.1 模型不需要理解整本书

图片生成阶段的目标不是让模型重新理解完整故事，而是：

```text
把当前页画对，并保持少数关键视觉元素不变。
```

因此，图片生成 prompt 不应默认包含完整故事全文。完整故事上下文应主要用于：

- 文本生成
- 分镜生成
- 封面总结
- 必要时生成前后页摘要

正文页图片生成阶段只需要：

- 当前页视觉摘要
- 当前页分镜
- 当前页相关视觉锚点
- 锁定画风
- 少量前后页连续性信息

当前页故事原文继续用于页面展示、编辑和导出；图片生成阶段不直接使用原文，而使用分镜生成阶段产出的 `summary`。

### 2.2 少量锚点比大量设定更可靠

不要把所有角色、物品、场景都塞进 prompt。每页只传本页真正出现、且需要保持一致的视觉锚点。

建议限制：

| 类型 | 限制 |
|---|---|
| 每本主角锚点 | 最多 3 个 |
| 每本关键物锚点 | 最多 5 个 |
| 每本核心场景锚点 | 最多 3 个 |
| 每页注入锚点 | 最多 3-5 个 |
| 每页参考图 | 最多 3 类 |
| 单个锚点描述 | 中文 40 字以内，英文 20-30 词以内 |

### 2.3 未设定但会重复出现的元素必须定型

如果故事里只是说“杯子”，但杯子会重复出现，系统不能保持 unknown。否则模型每次都会自由发挥，导致前后不一致。

对于重复出现或剧情关键的元素，即使用户没有给颜色、材质、形状，系统也应该主动生成一个稳定视觉设定，并在后续页面锁定。

示例：

```json
{
  "id": "cup_01",
  "name": "杯子",
  "visual_anchor": "红色陶瓷小杯子，圆形杯耳，一个白色星星图案",
  "locked_attributes": ["红色", "陶瓷", "圆形杯耳", "白色星星图案"],
  "reason": "多页出现的剧情关键物"
}
```

这不是模型推断事实，而是系统为了连续性做出的设计决策。

### 2.4 连续性是最高优先级

绘本是连续叙事，跨页视觉连续性必须是最高优先级。

连续性包含：

- 同一角色的外观、服装、配饰、比例、气质保持一致。
- 同一关键物的颜色、材质、形状、标志性细节保持一致。
- 同一核心场景的空间关系和主要视觉特征保持一致。
- 同一本书的整体画风、媒介质感、线条、色彩体系保持一致。

上一页图片是当前阶段最直接、最重要的连续性参考。后续引入视觉锚点后，连续性应由“上一页图片 + 视觉锚点 + 锁定画风”共同约束。

如果第一页风格偏了，或者杯子颜色错了，确实可能被连续性继续放大。这个风险不能通过降低连续性优先级解决，而应该通过锚点、QA 和用户修正来处理。

图片生成时应明确优先级：

```text
1. 跨页视觉连续性
2. 视觉锚点
3. 锁定画风
4. 当前页分镜
```

### 2.5 画风图片为主，短文本定边界

风格主图片比长描述更适合表达：

- 笔触和媒介质感。
- 色彩关系。
- 角色圆润程度。
- 背景复杂度。
- 整体氛围。

因此，画风系统不应依赖很长的 `style_description`。推荐策略是：

```text
风格主图片负责在第一页表达“主要像什么”。
第二页及以后不再重复上传画风图片，而通过上一页图片延续已经建立的画风。
短文本负责说明“学什么、不学什么”。
```

画风文本应尽量短、硬、少歧义：

```text
学习参考图的水彩质感、柔和边缘、低细节背景、温暖色彩。
不要复制参考图中的具体人物、构图、文字、场景。
不要生成写实、3D、动漫赛璐璐风格。
```

这样可以减少模型在长画风描述中抓错重点，也能避免文字描述和参考图互相竞争。

## 3. 视觉锚点

### 3.1 视觉锚点定义

视觉锚点是绘本中需要跨页保持一致的少量关键视觉对象。

类型包括：

- 主角：人物、动物、长期出现的角色。
- 关键物：杯子、书包、玩具熊、钥匙、魔法石等剧情物品。
- 核心场景：家、厨房、森林小屋、教室等反复出现的主要地点。
- 风格锚点：当前绘本锁定的图片风格版本。

### 3.2 锚点生成时机

推荐在分镜生成完成后，正式图片生成前，增加一个后台步骤：

```text
故事 + 分镜
-> 抽取重复出现实体
-> 选择少量视觉锚点
-> 对缺失属性做稳定定型
-> 给每页绑定本页相关锚点
-> 开始逐页生成图片
```

用户侧不需要进入单独的维护流程。普通用户只看到类似状态：

```text
正在统一角色与关键物设定...
```

### 3.3 锚点选择规则

满足以下任一条件的实体，可以成为锚点：

- 出现 2 次以上。
- 是主角或主要配角。
- 被角色持有、寻找、赠送、使用。
- 出现在标题、封面描述或故事核心冲突中。
- 视觉变化会明显破坏阅读体验。

不建议成为锚点：

- 一次性背景物。
- 普通花草、云朵、路人。
- 不影响剧情理解的随机装饰。
- 每页都会变化的环境细节。

### 3.4 锚点与分镜的关系

锚点应该进入分镜，但不要把完整锚点定义重复写进每一页分镜。

推荐两层结构：

1. 绘本级全局锚点表  
   定义锚点是谁、长什么样、哪些属性必须锁定。

2. 页面级分镜引用  
   每页只通过 `anchor_refs` 引用本页出现的锚点 ID。

全局锚点表示例：

```json
{
  "anchors": [
    {
      "id": "bear_01",
      "name": "小熊",
      "type": "character",
      "description": "棕色小熊儿童，圆脸，蓝色围巾，温和表情",
      "locked_attributes": ["棕色毛发", "圆脸", "蓝色围巾"]
    },
    {
      "id": "cup_01",
      "name": "红色杯子",
      "type": "object",
      "description": "红色陶瓷小杯子，圆形杯耳，一个白色星星图案",
      "locked_attributes": ["红色", "陶瓷", "圆形杯耳", "白色星星图案"]
    }
  ]
}
```

页面分镜中只引用：

```json
{
  "text": "小熊把杯子放在窗边。",
  "storyboard": {
    "visual_brief": "小熊在阳光照进来的厨房窗边，把杯子放到桌上。",
    "anchor_refs": ["bear_01", "cup_01"],
    "must_include": ["cup_01 清晰可见", "窗边桌子"],
    "composition": "中景，cup_01 在前景，bear_01 在画面中部",
    "avoid": ["不要出现文字", "不要改变 cup_01 的颜色和形状"]
  }
}
```

这样有三个好处：

- 分镜更短，不会重复塞锚点描述。
- 用户或系统修改全局锚点后，所有引用页面自动使用新定义。
- 每页图片生成时，只根据 `anchor_refs` 注入本页相关锚点。

## 4. 弱结构化分镜

### 4.1 当前分镜字段的问题

当前分镜通常包含：

```json
{
  "scene": "...",
  "characters": "...",
  "shot": "...",
  "color": "...",
  "lighting": "..."
}
```

这套结构的优点是稳定、可编辑、可落库，但问题是：

- 每页都强制生成 `color` 和 `lighting`，容易与锁定画风冲突。
- 字段过细时，模型会把分镜当成强视觉指令，削弱风格参考图。
- `characters` 只描述动作时，无法明确绑定全局锚点。
- 图片生成时容易同时收到“画风要求”和“分镜色彩光影要求”两套目标。

### 4.2 推荐分镜结构

不建议完全改成无格式文本。无格式文本不利于编辑、锚点绑定、局部重生成和连续性检查。

推荐改成弱结构化：

```json
{
  "text": "小熊把杯子放在窗边，阳光照进厨房。",
  "storyboard": {
    "summary": "小熊把红色杯子放在阳光照进来的厨房窗边，杯子清晰可见。",
    "visual_brief": "小熊在温暖厨房的窗边，把杯子轻轻放到桌上。",
    "anchor_refs": ["bear_01", "cup_01"],
    "must_include": ["cup_01 清晰可见", "窗边桌子"],
    "composition": "中景，cup_01 在前景，bear_01 在画面中部偏右",
    "avoid": ["不要出现文字", "不要改变 cup_01 的颜色和形状"]
  }
}
```

字段建议：

| 字段 | 用途 |
|---|---|
| `summary` | 当前页视觉摘要，用于图片生成，不直接使用故事原文 |
| `visual_brief` | 一段自然语言画面说明，替代过细的 scene/characters/color/lighting |
| `anchor_refs` | 当前页出现的锚点 ID |
| `must_include` | 本页必须出现的关键视觉信息 |
| `composition` | 简短构图要求 |
| `avoid` | 当前页禁止项 |

`color` 和 `lighting` 不应作为默认分镜字段。新生成和重生成分镜时不再产出这两个字段。

如果故事明确要求特殊色彩或光影，应写入 `visual_brief` 或当前页 `must_include`，不要再通过固定字段默认传入。

## 5. 图片生成 Prompt 结构

### 5.1 Prompt 不再默认包含完整故事上下文

旧结构倾向于：

```text
完整故事上下文
当前任务
当前页故事文本
当前页分镜
风格描述
参考图说明
```

推荐改为：

```text
当前页任务
当前页视觉摘要
当前页分镜
锁定画风
本页视觉锚点
硬性连续性规则
必要的前后页摘要
禁止项
```

完整故事上下文如果确实需要，可以压缩为很短的 narrative note，而不是逐页全文。

### 5.2 Gemini 图片生成示例

System instruction：

```text
You are a professional children's picture book illustrator.

Your priority order is strict:
1. Preserve cross-page visual continuity.
2. Keep VISUAL ANCHORS exactly consistent.
3. Follow LOCKED ART STYLE.
4. Follow the CURRENT PAGE storyboard.

Do not invent or change locked attributes of visual anchors.
If the current page description or mood conflicts with locked style or visual anchors, follow locked style and visual anchors.
If a previous page image is provided, use it as the strongest continuity reference for established character appearance, object appearance, art style, palette, and scene continuity.
Do not change established visual identity unless the user explicitly asks for a change.

Create one full-bleed children's book illustration.
No text, letters, captions, labels, borders, or frames.
```

User prompt：

```text
CURRENT PAGE:
Page 3 of 10

VISUAL SUMMARY:
小熊把红色杯子放在阳光照进来的厨房窗边，杯子需要清晰可见。

CURRENT STORYBOARD:
- Visual brief: little bear gently places the cup on a table beside a sunny kitchen window
- Anchor refs: bear_01, cup_01
- Must include: cup_01 clearly visible; table beside the window
- Composition: medium shot, cup_01 in the foreground, bear_01 near the center-right

LOCKED ART STYLE:
Style name: 柔和水彩绘本
Style reference guidance: learn watercolor texture, soft edges, paper grain, warm bright palette, low-detail background, child-friendly rounded shapes from the provided style reference images
Do not copy specific subjects, scenes, text, or composition from style reference images
Negative prompt: photorealistic, 3D render, anime, comic panel, dark cinematic lighting, sharp glossy texture, realistic photography

STYLE MAIN REFERENCE IMAGE:
Only when generating page 1, use the provided style main image as the primary locked art style reference.
Learn its medium, brushwork, line quality, palette, texture, background detail level, and overall children's book feeling.
Do not copy its specific subject, character, scene, text, or composition.

VISUAL ANCHORS FOR THIS PAGE:
- bear_01: small brown bear child, round face, blue scarf, gentle expression
- cup_01: small red ceramic cup, round handle, one white star decal

HARD CONTINUITY RULES:
- bear_01 must keep the same brown fur, round face, and blue scarf.
- cup_01 must stay red. Do not change it to green, blue, glass, metal, or a different shape.
- cup_01 must keep the round handle and white star decal.
- Only vary pose, angle, expression, and lighting.

PAGE GOAL:
Show bear_01 placing cup_01 near the sunny kitchen window.
Make cup_01 clearly recognizable.
```

Gemini parts 中参考图建议按用途拆开说明：

```text
1. STYLE MAIN REFERENCE IMAGE
Only for page 1. Use as the primary locked art style reference. Learn only medium, brushwork, line quality, palette, texture, background detail level, and overall children's book feeling. Do not copy subjects.

2. CHARACTER REFERENCE IMAGES
Use only to preserve person or character identity and recognizability. Do not learn art style, palette, background, or composition from these images.

3. ANCHOR REFERENCE IMAGES
Use these to preserve exact appearance of visual anchors.

4. PREVIOUS PAGE IMAGE
Use as the strongest continuity reference for established character appearance, object appearance, art style, palette, and scene continuity.
```

### 5.3 豆包图片生成示例

豆包侧建议使用中文强约束：

```text
你是一名专业儿童绘本插画师。

【优先级规则，必须严格遵守】
1. 第一优先级：跨页视觉连续性
2. 第二优先级：视觉锚点一致性
3. 第三优先级：锁定画风
4. 第四优先级：当前页分镜

如果当前页描述或氛围与锁定画风、视觉锚点冲突，以锁定画风和视觉锚点为准。
如果提供了上一页图片，上一页图片是最重要的连续性参考。必须尽量保持已建立的角色外观、关键物外观、画风、色彩体系和场景延续。
除非用户明确要求改变，否则不要改变已经建立的视觉身份。

【当前页】
第3页 / 共10页

【当前页视觉摘要】
小熊把红色杯子放在阳光照进来的厨房窗边，杯子需要清晰可见。

【当前页分镜】
- 画面简述：小熊在阳光照进来的厨房窗边，把杯子轻轻放到桌上
- 锚点引用：bear_01、cup_01
- 必须出现：cup_01 清晰可见、窗边桌子
- 构图：中景，cup_01 在前景，bear_01 在画面中部偏右

【锁定画风】
画风名称：柔和水彩绘本
画风参考要求：第一页从风格主图片中学习水彩质感、柔软边缘、纸张纹理、温暖明亮色彩、低细节背景、儿童友好的圆润造型；后续页面通过上一页图片延续画风
不要复制风格主图片中的具体人物、场景、文字或构图
负面提示词：写实照片、3D渲染、动漫赛璐璐、漫画分格、黑暗电影感、锐利高光、真实摄影质感

【风格主图片参考】
仅第一页生成时提供风格主图片。风格主图片是锁定画风的主要参考，请学习它的媒介质感、笔触、线条、色彩关系、背景细节密度和儿童绘本整体气质。
不要复制风格主图片中的具体人物、场景、构图、文字或物体。

【本页视觉锚点】
- bear_01：棕色小熊儿童，圆脸，蓝色围巾，温和表情
- cup_01：红色陶瓷小杯子，圆形杯耳，一个白色星星图案

【硬性连续性要求】
- bear_01 必须保持棕色毛发、圆脸、蓝色围巾
- cup_01 必须保持红色，禁止变成绿色、蓝色、玻璃杯、金属杯或其他形状
- cup_01 必须保留圆形杯耳和白色星星图案
- 只允许变化姿势、角度、表情、光影

【画面目标】
生成一张儿童绘本内页插画：bear_01 正在把 cup_01 放到窗边桌上，cup_01 必须清晰可辨。

【禁止】
图片中不要出现任何文字、字母、标题、标签、边框。
```

豆包参考图说明建议：

```text
【第一页参考图片说明】
第1张：风格参考图片，也就是风格主图片。只学习画风、媒介质感、色彩、线条、背景细节密度和氛围。
第2到第N张：角色参考图片。只用于保持角色身份和角色外观可辨识，不学习画风、色调、背景复杂度或构图。角色不一定是人。

【第2到N页参考图片说明】
第1张：连续性参考图片，也就是上一页插画。用于保持已建立的角色外观、关键物外观、画风、色彩体系和场景延续。
第2到第N张：角色参考图片。只用于保持角色身份和角色外观可辨识，不学习画风、色调、背景复杂度或构图。角色不一定是人。
```

## 6. 参考图策略

参考图不是越多越好。参考图过多时，模型会在风格、角色、上一页、物品之间竞争注意力。

图片生成阶段输入给大模型的图片只分三类：

| 类型 | 含义 | 使用时机 |
|---|---|---|
| 风格参考图片 | 风格主图片，只用于建立整本绘本的锁定画风 | 只在创建第一页时传入 |
| 连续性参考图片 | 上一页图片，用于保持跨页视觉连续性 | 第二页及以后传入 |
| 角色参考图片 | 用户上传或系统选取的角色参考图，角色不一定是人，也可以是动物、玩偶等主要角色 | 第一页及后续页面都可传入 |

建议使用以下参考图：

1. 风格主图片  
   只在第一页传入，用于建立整本绘本的锁定画风。第二页及以后不再传入画风图片。

2. 角色参考图  
   现有引用参考图片统一视为角色参考图片。角色不一定是人，也可以是动物、玩偶、拟人化物体等主要角色。角色参考图只用于保持角色身份、外观和可辨识性，不用于学习画风。

3. 锚点参考图  
   用于保持角色或关键物外观一致。

4. 连续性参考图  
   用于叙事连续性，是最高优先级连续性参考。

推荐顺序：

```text
第一页：风格参考图片 -> 角色参考图片
第二页及以后：连续性参考图片 -> 角色参考图片
```

禁止组合：

```text
第一页不传连续性参考图片，因为第一页没有上一页。
第2到N页不传风格参考图片，因为画风应由上一页连续性图片延续。
角色参考图片两种场景都可以传。
```

风格主图片来源：

```text
优先使用 image_style_version.reference_images 中 is_cover=true 的图片。
如果没有 is_cover=true，则使用排序后的第一张画风参考图。
其他画风参考图本阶段不传入图片生成 prompt。
```

如果某页锚点很多，优先传主角和剧情关键物。不要为了“完整”把所有参考图都传进去。

## 8. 检查与自动优化

### 8.1 不引入智能体的原则

检查和自动优化不需要先引入智能体。推荐把模型能力包装成一组窄任务 service，由后端代码控制流程：

```text
生成页面图片
-> 结构化检查
-> 生成修复指令
-> 自动修复或重生成
-> 再检查
-> 通过则保存，失败则记录问题
```

模型只负责完成小任务：

- 判断指定锚点是否符合要求。
- 判断画风是否明显跑偏。
- 判断本页必须出现的内容是否出现。
- 输出结构化修复指令。

流程控制、重试次数、状态更新、成本控制都由后端 service 负责。

### 8.2 建议新增 service

建议新增：

```text
app/services/visual_quality_service.py
```

核心函数：

```python
async def check_style_compliance(...): ...
async def check_anchor_consistency(...): ...
async def check_page_requirements(...): ...
async def build_repair_instruction(...): ...
async def run_page_quality_check(...): ...
```

职责边界：

| 函数 | 职责 |
|---|---|
| `check_style_compliance` | 检查图片是否明显偏离锁定画风 |
| `check_anchor_consistency` | 检查本页 `anchor_refs` 对应的锚点是否存在且属性一致 |
| `check_page_requirements` | 检查 `must_include` 是否满足，是否出现禁止内容 |
| `build_repair_instruction` | 把检查问题转成图片编辑或重生成指令 |
| `run_page_quality_check` | 汇总多个检查结果，返回统一 QA 结果 |

### 8.2.1 模型调用策略

上面的函数是代码职责划分，不等于每个函数都必须单独调用一次模型。

首版应优先控制成本和耗时：

```text
每页图片生成后
-> 调用 1 次 check_anchor_consistency
-> 失败时调用 1 次 edit_image
-> 再调用 1 次 check_anchor_consistency 复查
-> 仍失败则记录 warning
```

也就是说，最小版本每页正常情况下只增加 1 次视觉检查模型调用；只有检查失败并触发修复时，才增加 1 次图片编辑调用和 1 次复查调用。

后续如果加入画风检查和页面要求检查，建议优先合并到 `run_page_quality_check` 的一次模型调用中完成：

```text
run_page_quality_check
-> 同一次视觉理解调用内检查 style、anchors、must_include、avoid
-> 返回统一 issues JSON
```

除非某类检查误判明显、需要单独调优，否则不要把 `check_style_compliance`、`check_anchor_consistency`、`check_page_requirements` 拆成 3 次独立模型调用。否则每页会重复解析同一张图片，成本和耗时都会明显增加。

### 8.3 检查输入

检查函数不要读取整本故事，只读取当前页必要信息：

```json
{
  "storybook_id": 12,
  "page_id": 34,
  "page_index": 3,
  "image_url": "...",
  "style": {
    "style_name": "柔和水彩绘本",
    "reference_guidance": "学习水彩质感、柔和边缘、纸张纹理、温暖明亮色彩、低细节背景",
    "negative_prompt": "写实照片、3D渲染、动漫赛璐璐"
  },
  "storyboard": {
    "visual_brief": "小熊在阳光照进来的厨房窗边，把杯子轻轻放到桌上",
    "anchor_refs": ["bear_01", "cup_01"],
    "must_include": ["cup_01 清晰可见", "窗边桌子"],
    "composition": "中景，cup_01 在前景，bear_01 在画面中部偏右",
    "avoid": ["不要出现文字", "不要改变 cup_01 的颜色和形状"]
  },
  "anchors": [
    {
      "id": "bear_01",
      "type": "character",
      "description": "棕色小熊儿童，圆脸，蓝色围巾，温和表情",
      "locked_attributes": ["棕色毛发", "圆脸", "蓝色围巾"]
    },
    {
      "id": "cup_01",
      "type": "object",
      "description": "红色陶瓷小杯子，圆形杯耳，一个白色星星图案",
      "locked_attributes": ["红色", "圆形杯耳", "白色星星图案"]
    }
  ]
}
```

### 8.4 检查输出

所有检查都返回结构化 JSON，禁止返回自由文本：

```json
{
  "passed": false,
  "severity": "high",
  "issues": [
    {
      "type": "anchor_mismatch",
      "anchor_id": "cup_01",
      "expected": "红色陶瓷小杯子，圆形杯耳，一个白色星星图案",
      "actual": "杯子看起来是绿色，星星图案不明显",
      "severity": "high",
      "repair_strategy": "edit",
      "repair_instruction": "将 cup_01 改为红色陶瓷小杯子，保留圆形杯耳和白色星星图案。其他角色、背景、构图和画风保持不变。"
    }
  ]
}
```

`repair_strategy` 建议枚举：

| 值 | 含义 |
|---|---|
| `none` | 不需要修复 |
| `edit` | 局部编辑图片 |
| `regenerate` | 整页重生成 |
| `manual_review` | 自动修复风险高，交给用户 |

### 8.5 检查 Prompt 示例

检查 prompt 必须非常窄，不问“图片好不好”，只问指定事实是否成立。

锚点检查示例：

```text
你是绘本图片质检器。
只检查指定视觉锚点是否满足要求，不评价其他内容。

图片中应包含：
- cup_01：红色陶瓷小杯子，圆形杯耳，一个白色星星图案

检查要求：
1. cup_01 是否存在。
2. cup_01 是否为红色。
3. cup_01 是否保留圆形杯耳。
4. cup_01 是否保留白色星星图案。

请只返回 JSON：
{
  "passed": true/false,
  "issues": [
    {
      "type": "anchor_missing|anchor_mismatch",
      "anchor_id": "cup_01",
      "expected": "...",
      "actual": "...",
      "severity": "low|medium|high",
      "repair_strategy": "edit|regenerate|manual_review",
      "repair_instruction": "..."
    }
  ]
}

不要输出 JSON 以外的内容。
```

画风检查示例：

```text
你是绘本画风质检器。
只判断图片是否明显偏离锁定画风，不评价构图和故事内容。

锁定画风：
- 水彩质感
- 柔和边缘
- 纸张纹理
- 温暖明亮色彩
- 低细节背景
- 儿童友好的圆润造型

禁止风格：
- 写实照片
- 3D渲染
- 动漫赛璐璐
- 黑暗电影感

请只返回 JSON：
{
  "passed": true/false,
  "issues": [
    {
      "type": "style_mismatch",
      "expected": "...",
      "actual": "...",
      "severity": "low|medium|high",
      "repair_strategy": "regenerate|manual_review",
      "repair_instruction": "..."
    }
  ]
}
```

### 8.6 自动修复策略

检查失败后不要一律重生成。先判断问题类型。

适合局部编辑的问题：

- 杯子颜色错。
- 衣服颜色错。
- 漏了星星图案。
- 主角围巾颜色错。
- 关键物形状轻微不一致。

局部编辑指令示例：

```text
将 cup_01 改为红色陶瓷小杯子，保留圆形杯耳和白色星星图案。
其他角色、背景、构图、光影和画风保持不变。
图片中不要添加文字。
```

适合整页重生成的问题：

- 画风完全错。
- 主角不存在。
- 当前页主要场景错了。
- 构图完全不符合 `must_include`。
- 参考图或锚点误读严重。

重生成时应把失败原因注入 prompt，但仍然保持简短：

```text
上一次生成失败原因：
- cup_01 被生成成绿色。
- cup_01 必须是红色陶瓷小杯子，带圆形杯耳和白色星星图案。

本次必须修正以上问题。
```

### 8.7 有限修复循环

不要无限自动优化。建议每页最多：

```text
首次生成
-> 检查
-> 局部修复 1 次
-> 再检查
-> 必要时整页重生成 1 次
-> 再检查
-> 仍失败则保存 warning，交给用户
```

推荐限制：

| 项 | 限制 |
|---|---|
| 每页局部编辑 | 最多 1 次 |
| 每页整页重生成 | 最多 1 次 |
| 每页 QA 检查 | 最多 3 次 |
| 高风险问题 | 不自动修，进入 manual_review |

这样成本可控，也能避免模型越修越偏。

### 8.8 页面状态

建议页面生成过程增加 QA 状态：

```text
pending
generating
checking
repairing
finished
finished_with_warnings
error
```

QA 结果可以存为 JSON：

```json
{
  "qa_status": "finished_with_warnings",
  "qa_attempts": 2,
  "issues": [
    {
      "type": "anchor_mismatch",
      "anchor_id": "cup_01",
      "severity": "medium",
      "message": "cup_01 星星图案不明显",
      "repair_strategy": "manual_review"
    }
  ]
}
```

首版可以不阻塞整本生成。即使某页 QA 失败，也可以保存页面并标记 warning，让用户后续在编辑器里处理。

### 8.9 最小可落地版本

第一版只做最关键闭环：

1. 图片生成后调用 1 次 `check_anchor_consistency`。
2. 只检查本页 `anchor_refs`。
3. 检查失败时生成 `repair_instruction`。
4. 调用一次 `edit_image`。
5. 再调用 1 次 `check_anchor_consistency` 复查。
6. 仍失败则记录 warning，不阻塞整本生成。

第一版不默认调用 `check_style_compliance` 和 `check_page_requirements`。这两类检查可以先保留函数接口，等锚点一致性闭环验证稳定后再接入。

这已经能覆盖“红杯变绿杯”“围巾颜色变了”“关键物不见了”等高频连续性问题。

## 9. 分阶段实施

### 9.1 第一阶段：上下文收敛

目标：减少完整故事上下文对图片生成的干扰。

工作项：

- 调整 Gemini 分镜生成 prompt，不再生成 `color`、`lighting`。
- 调整豆包分镜生成 schema 和 prompt，不再生成 `color`、`lighting`。
- 调整单页分镜重生成 prompt，不再生成 `color`、`lighting`。
- 调整 Gemini 分镜生成 prompt，每页生成 `storyboard.summary`。
- 调整豆包分镜生成 schema 和 prompt，每页生成 `storyboard.summary`。
- 调整 Gemini 图片生成 prompt。
- 调整豆包图片生成 prompt。
- 仅在第一页将风格主图片作为唯一画风图片传入图片生成 prompt。
- 将现有引用参考图片明确为角色参考图片。
- 默认不传完整故事全文。
- 图片生成使用当前页 `storyboard.summary`，不直接使用故事原文。
- 只保留当前页摘要、当前页分镜、锁定风格。
- 上一页图片强化为最高优先级连续性参考。

验收：

- 同一风格下生成结果更接近锁定画风。
- 当前页主体更贴近分镜。
- 前后页不会因为第一页错误风格被持续放大。
- 新生成和重生成分镜不再包含默认 `color`、`lighting` 字段。
- 新生成和重生成分镜都包含 `storyboard.summary`。
- 风格主图片只在第一页作为画风参考进入图片生成 prompt。
- 现有引用参考图片会以角色参考图片的身份进入 prompt。

### 9.2 第二阶段：弱结构化分镜与锚点引用

目标：让分镜承载锚点引用，但不重复锚点定义。

工作项：

- 新增弱结构化分镜字段。
- 分镜中使用 `anchor_refs` 引用全局锚点。
- 继续保持不生成默认 `color`、`lighting` 字段。
- 图片生成时根据 `anchor_refs` 注入对应锚点定义。

验收：

- 分镜更短，图片 prompt 更聚焦。
- 风格主图片不再被每页 color/lighting 字段明显干扰。
- 锚点可以稳定进入图片生成链路。

### 9.3 第三阶段：视觉锚点

目标：控制少数关键元素的跨页一致性。

工作项：

- 从故事和分镜中抽取重复出现实体。
- 生成最多 8 个视觉锚点。
- 给每页绑定本页相关锚点。
- 图片生成时只注入本页锚点。

验收：

- 重复出现的杯子、书包、衣服颜色更稳定。
- 不需要用户手动维护锚点。
- 每页 prompt 长度可控。

### 9.4 第四阶段：检查与自动修复

目标：生成后自动检查关键锚点和画风，并做有限自动修复。

工作项：

- 新增 `visual_quality_service.py`。
- 实现 `check_anchor_consistency`。
- 实现 QA JSON 输出契约。
- 生成失败时调用一次 `edit_image`。
- 页面保存 QA 状态和 issues。

验收：

- 能识别关键物颜色变化。
- 能对局部错误自动生成修复指令。
- 修复失败时不阻塞整本绘本生成。
- 页面能展示 warning，方便用户后续处理。

### 9.5 第五阶段：锚点参考图与更完整检查

目标：从文字约束升级到图片约束和生成后检查。

工作项：

- 首次成功出现锚点后，沉淀锚点参考图。
- 后续页面按需传入锚点参考图。
- 生成后检查锚点是否跑偏。
- 记录连续性问题，必要时提示用户一键修正。

验收：

- 主角、关键物跨页一致性明显提升。
- 系统能指出具体不一致问题。
- 用户不需要理解 prompt，也能修正明显错误。

## 10. 风险与边界

### 10.1 视觉锚点不是越多越好

锚点过多会重新制造注意力分散问题。首版宁可少管，也不要把每个背景物都管起来。

### 10.2 自动定型可能不符合用户偏好

如果用户没有指定杯子颜色，系统自动定为红色，可能与用户想象不同。

处理方式：

- 默认自动定型，不阻塞用户。
- 生成后用户可通过编辑或重新生成改变锚点。
- 如果用户修改锚点，后续页面同步使用新锚点。

### 10.3 最高优先级连续性仍然可能放大错误

连续性是最高优先级，因此上一页图片中的错误确实可能被后续页面延续。

处理方式：

- 有上一页图片时默认传入上一页图，保证连续性。
- 如果上一页被标记为风格或锚点错误，不作为后续参考。
- 如果上一页被标记为错误，使用最近一张通过 QA 或用户确认的页面作为连续性参考。
- 锚点参考图用于修正上一页中已经确认的错误，并继续服务于跨页连续性。

### 10.4 风格主图片可能被误读

风格主图片可能被模型误读为角色、场景或构图参考。

处理方式：

- 风格主图片必须配短文本说明“只学习画风，不复制主体”。
- 风格图片和锚点图片分组传入，不混在同一段说明里。
- 风格描述保持短，避免与图片表达互相冲突。

### 10.5 检查模型也会误判

检查本身也是模型输出，可能误判。

处理方式：

- 检查 prompt 保持窄任务，只检查明确事实。
- 只对高置信、高影响问题自动修复。
- 中低置信问题只记录 warning。
- 自动修复次数必须有限。
- 保留原图、修复图和 QA issue，方便回滚和人工判断。

## 11. 总结

本方案的核心不是让 prompt 更长，而是让 prompt 更窄、更硬、更贴近当前页。

最终策略：

```text
不让模型理解整本书。
只让模型画好当前页。
只锁定少数关键视觉锚点。
只传当前页真正需要的上下文。
上一页图片作为最高优先级连续性参考。
```

这样更符合生成模型的注意力特点，也更容易在 Gemini 和豆包两条链路上持续调优。
