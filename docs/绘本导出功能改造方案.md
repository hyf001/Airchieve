# 绘本导出功能改造方案

## 1. 背景

当前前端下载逻辑集中在 `frontend/src/services/storybookService.ts` 的 `downloadStorybookImage` 中。它在浏览器端使用 Canvas 将每页图片重新绘制成 PDF 页面，并额外合成了：

- 底部渐变遮罩
- `page.text` 故事正文白字
- 水印
- 所有页面页码

这套逻辑已经不符合当前编辑器的图层化数据模型。现在页面内容应由“底图 + 图层”决定，导出结果需要忠实反映用户在编辑器中看到和编辑的页面图层，而不是重新套一套固定版式。

## 2. 改造目标

### 2.1 通用目标

导出时每一页都应先渲染为一张“页面内容图”：

1. 绘制页面底图 `page.image_url`
2. 按 `layer_index` 顺序合成所有 `visible=true` 的图层
3. 不绘制底部渐变遮罩
4. 不自动把 `page.text` 故事正文合成到图片上
5. 不绘制水印
6. 封面和封底不显示页码

注意：这里的“不合成文字”指不再把 `page.text` 作为固定字幕画到底部。用户通过文字工具创建的文字图层属于图层内容，仍然需要参与导出合成。

### 2.2 支持格式

导出功能支持三种格式：

- PDF
- PNG 长图
- JPG 长图

PDF 面向打印和分页阅读；PNG/JPG 面向分享、预览和社交传播。

## 3. 产品交互方案

### 3.1 下载弹窗改造

将当前 `DownloadDialog` 从“是否添加水印”弹窗改造为“导出作品”设置弹窗。

弹窗第一项为导出格式：

- PDF
- PNG 长图
- JPG 长图

不同格式展示不同配置项。

### 3.2 PDF 配置项

PDF 支持纸张设置和打印预览。

配置项：

- 纸张尺寸
  - 原图尺寸
  - A4
  - A5
  - 方形 210mm x 210mm
- 纸张方向
  - 自动
  - 竖版
  - 横版
- 图片适配方式
  - 完整显示：保留完整画面，可能出现留白
  - 铺满裁切：铺满整张纸，可能裁掉边缘
- 打印安全边距
  - 默认关闭
  - 开启后内容不会贴边，适合普通家用打印机
- 正文页页码
  - 默认开启
  - 只对正文页生效
  - 封面 `cover` 和封底 `back_cover` 永远不显示页码

### 3.3 PNG/JPG 配置项

PNG 和 JPG 不提供纸张设置，固定导出为原始页面尺寸的长图。

规则：

- 使用绘本原始页面尺寸
- 从上到下拼接所有页面
- 不显示纸张尺寸、横竖版、适配方式、安全边距配置
- 不添加页码
- JPG 额外提供图片质量选项
  - 高质量，默认 0.92
  - 标准质量，可选 0.82
  - 小体积，可选 0.72

## 4. 预览方案

导出弹窗需要提供预览能力，让用户在导出前理解所选设置的实际效果。

### 4.1 PDF 预览

PDF 预览展示“纸张上的打印效果”。

预览内容：

- 纸张轮廓，比例与所选纸张一致
- 当前页面图片在纸张中的位置和大小
- `contain` 模式下的留白
- `cover` 模式下的裁切效果
- 正文页页码位置
- 封面和封底无页码

预览页切换只提供：

- 上一页
- 下一页

默认预览当前编辑页。切换到第一页或最后一页时，上一页或下一页按钮置灰。

### 4.2 PNG/JPG 长图预览

PNG/JPG 预览展示长图缩略效果。

预览内容：

- 所有页面按原比例纵向堆叠
- 页面之间不额外添加分隔线或间距，预览应与最终长图一致
- 展示预计输出尺寸
  - 单页尺寸：`pageWidth x pageHeight`
  - 长图尺寸：`pageWidth x pageHeight * pageCount`
- 如果长图高度超过浏览器 Canvas 安全上限，提示将自动分卷导出

### 4.3 预览性能策略

预览不需要立即做高清 Canvas 合成。

建议：

- 预览阶段使用页面图片 URL 和现有图层 DOM 预览方式快速展示
- 点击导出后再执行高清 Canvas 合成
- 这样可以保证弹窗打开和切换设置时响应足够快

注意：预览层允许使用低成本 DOM 渲染，但尺寸、留白、裁切、页码和长图拼接规则必须与最终导出一致，避免用户看到的预览和下载结果不一致。

## 5. 数据结构设计

建议新增导出配置类型。

```ts
export type ExportFormat = 'pdf' | 'png' | 'jpg';
export type PaperSize = 'original' | 'a4' | 'a5' | 'square-210';
export type PaperOrientation = 'auto' | 'portrait' | 'landscape';
export type FitMode = 'contain' | 'cover';

export interface ExportOptions {
  format: ExportFormat;

  // PDF only
  paperSize: PaperSize;
  orientation: PaperOrientation;
  fitMode: FitMode;
  safeMargin: boolean;
  showPageNumbers: boolean;
  dpi: number;

  // JPG only
  jpgQuality: number;
}
```

默认值：

```ts
export const DEFAULT_EXPORT_OPTIONS: ExportOptions = {
  format: 'pdf',
  paperSize: 'original',
  orientation: 'auto',
  fitMode: 'contain',
  safeMargin: false,
  showPageNumbers: true,
  dpi: 300,
  jpgQuality: 0.92,
};
```

PNG/JPG 虽然会携带完整配置对象，但只使用：

- `format`
- `jpgQuality`

其他 PDF 配置项在 PNG/JPG 模式下不展示、不生效。

## 6. 渲染管线设计

导出逻辑应拆成三层，避免 PDF、PNG、JPG 各自实现一套图层合成。

### 6.1 获取页面详情

导出时需要获取每一页的完整详情，包括图层数据。页面详情可以并发获取，但高清图片加载、Canvas 合成和 PDF 写入应采用逐页流水线，避免一次性持有所有高分辨率 Canvas。

```ts
const pageDetails = await Promise.all(
  pages.map(page => getPageDetail(page.id))
);
```

如果页面数量很多，后续也可以将详情获取改为有限并发，例如每次 3-5 页，进一步降低接口和内存压力。

下载前需要先提交当前正在编辑的文字图层，避免最后一次输入还未持久化：

```ts
textEditToolRef.current?.commitCurrentEdits();
```

### 6.2 页面内容合成

第一层渲染页面内容图。

```ts
renderPageArtwork(pageDetail, sourceImage): Promise<HTMLCanvasElement>
```

职责：

- 创建与原始页面图片同尺寸的 Canvas
- 绘制底图
- 合成所有可见图层
- 不处理纸张
- 不处理页码
- 不处理导出格式

由于同一本绘本内所有页面的分辨率和尺寸一致，导出时可以使用第一张有效图片的尺寸作为本次绘本的统一页面尺寸。不同绘本各自计算。

所有非同源图片 URL 在进入 Canvas 前都必须走同源代理或转换为可安全绘制的 URL，避免 Canvas 被 CORS 污染。该规则适用于页面底图、图片图层、贴纸图层，以及未来任何引用外部图片资源的图层。

### 6.3 PDF 页面排版

第二层仅用于 PDF。

```ts
renderPdfPage(artworkCanvas, pageDetail, options, contentPageNumber)
```

职责：

- 计算纸张尺寸
- 将页面内容图按 `contain` 或 `cover` 放入纸张
- 应用安全边距
- 给正文页绘制页码
- 跳过封面和封底页码

### 6.4 PNG/JPG 长图生成

第二层用于 PNG/JPG。

```ts
renderLongImage(artworkCanvases, format, jpgQuality)
```

职责：

- 创建长图 Canvas
- 宽度等于单页宽度
- 高度等于单页高度乘以页数
- 从上到下依次绘制每页内容图
- PNG 使用 `image/png`
- JPG 使用 `image/jpeg` 并应用 `jpgQuality`

JPG 导出前需要先填充白色背景，避免透明区域变黑：

```ts
ctx.fillStyle = '#ffffff';
ctx.fillRect(0, 0, canvas.width, canvas.height);
```

### 6.5 文件输出

第三层负责文件输出。

```ts
exportAsPdf(canvases, options)
exportAsPngLongImage(canvas)
exportAsJpgLongImage(canvas, quality)
```

PDF：

- 使用 `jspdf`
- 多页输出
- 文件名：`绘本名.pdf`
- PDF 导出应逐页渲染并立即写入 jsPDF。写入后释放当前页 Canvas 引用，不保留所有页面 Canvas。

PNG/JPG：

- 正常情况下输出单张长图
- 文件名：`绘本名.png` / `绘本名.jpg`
- 如果长图超过 Canvas 上限，则按分卷输出
- PNG/JPG 需要先完成每页内容图合成，再绘制到长图 Canvas。绘制完成后应尽快释放单页 Canvas 引用。

## 6.6 导出进度与取消

导出流程需要支持进度回调和取消。

建议导出服务接收运行时参数：

```ts
interface ExportRuntimeOptions {
  signal?: AbortSignal;
  onProgress?: (progress: {
    current: number;
    total: number;
    message?: string;
  }) => void;
}
```

进度上报建议：

- 获取页面详情完成后，上报准备完成
- 每完成一页高清合成后上报一次
- PDF 每写入一页后上报一次
- PNG/JPG 长图每绘制一页后上报一次
- 文件生成完成后上报 100%

取消机制：

- `EditorView` 或导出弹窗持有 `AbortController`
- 用户点击取消时调用 `abortController.abort()`
- 导出流水线在每个异步阶段和每页循环开始前检查 `signal.aborted`
- 图片加载、页面详情请求和渲染循环都应尽量响应取消
- 取消后清理临时 Canvas 引用并关闭进度状态

## 7. 纸张与打印适配规则

### 7.1 纸张尺寸

使用 `mm` 作为 PDF 尺寸单位。

```ts
const PAPER_SIZES = {
  a4: { width: 210, height: 297 },
  a5: { width: 148, height: 210 },
  'square-210': { width: 210, height: 210 },
};
```

`original` 使用图片像素和 DPI 换算：

```ts
widthMm = imageWidthPx / dpi * 25.4;
heightMm = imageHeightPx / dpi * 25.4;
```

### 7.1.1 DPI 作用范围

`dpi` 仅用于 PDF 的物理尺寸换算，不用于重新采样或放大图片。

明确规则：

- `paperSize === 'original'` 时，`dpi` 用于将图片像素换算为 PDF 页面毫米尺寸
- `paperSize === 'a4' / 'a5' / 'square-210'` 时，纸张物理尺寸固定，`dpi` 不决定 PDF 页面大小
- PDF 内嵌图片默认使用页面内容图的原始像素分辨率
- 不因为选择 A4/A5 就强制把每页渲染成 `2480 x 3508` 这类 300 DPI 位图
- 如后续需要“印刷级重采样”，应单独增加清晰度选项，并明确提示内存和导出耗时

这样可以避免 A4/A5 导出时为多页绘本创建大量超高分辨率 Canvas，导致浏览器内存压力过大。

### 7.2 自动方向

当 `orientation === 'auto'` 时：

- 图片比例更接近横版纸张时使用横版
- 图片比例更接近竖版纸张时使用竖版

```ts
imageRatio = imageWidth / imageHeight;
portraitRatio = paperShort / paperLong;
landscapeRatio = paperLong / paperShort;
```

选择与 `imageRatio` 差异更小的方向。

### 7.3 适配方式

完整显示 `contain`：

- 保留完整画面
- 可能出现上下或左右留白
- 适合不希望裁切内容的默认场景

铺满裁切 `cover`：

- 铺满整张纸
- 可能裁掉边缘
- 适合用户明确需要满版效果的场景

### 7.4 安全边距

安全边距只对 PDF 生效。

建议默认值：

- A4 / A5：5mm
- 方形：5mm
- 原图尺寸：不默认开启

开启后，图片适配区域从整张纸缩小为：

```ts
contentBox = {
  x: margin,
  y: margin,
  width: paperWidth - margin * 2,
  height: paperHeight - margin * 2,
};
```

## 8. 页码规则

PDF 页码规则：

- `showPageNumbers=false` 时不显示任何页码
- `page_type === 'cover'` 不显示页码
- `page_type === 'back_cover'` 不显示页码
- 只有正文页显示页码
- 正文页页码从 1 开始
- 封面不占用正文页页码
- 封底不占用正文页页码

PNG/JPG 长图：

- 不显示页码

## 9. 图层合成规则

需要支持现有图层类型：

- `text`
- `draw`
- `image`
- `sticker` 可后续按图片图层兼容
- `adjustment` 暂不应用，第一版导出时跳过

通用规则：

- 只渲染 `visible=true` 且 `content` 存在的图层
- 按 `layer_index` 从小到大渲染
- 不渲染被锁定状态不影响导出，`locked` 只影响编辑行为
- 未支持的图层类型应跳过并记录调试日志，不应中断整本绘本导出

### 9.1 文字图层

支持字段：

- `x`
- `y`
- `width`
- `height`
- `text`
- `fontFamily`
- `fontSize`
- `fontColor`
- `fontWeight`
- `textAlign`
- `lineHeight`
- `backgroundColor`
- `borderRadius`
- `rotation`

文字换行、垂直居中、背景圆角应尽量与编辑器预览保持一致。

Canvas 文字渲染依赖浏览器已加载的字体。导出前需要确保文字图层使用到的字体已经可用：

- 系统字体可以直接使用
- Web Font 或后续引入的自定义字体需要通过 `document.fonts.load(...)` 或 `FontFace.load()` 等待加载完成
- 如果字体加载失败，应 fallback 到默认字体并记录警告
- 否则 Canvas `fillText` 可能使用默认字体，导致导出排版与编辑器预览不一致

文字图层带 `rotation` 时，Canvas 绘制应使用 `save -> translate(center) -> rotate -> draw -> restore` 的方式，围绕图层中心点旋转，避免影响后续图层。

### 9.2 涂鸦图层

支持字段：

- `strokes`

实际结构：

```ts
interface DrawLayerContent {
  strokes: Array<{
    points: number[][];
    color: string;
    brushSize: number;
  }>;
}
```

`strokes` 是笔画数组，每一笔独立携带 `points`、`color` 和 `brushSize`。实现时不要把 `points/color/brushSize` 当作 `DrawLayerContent` 的平级字段。

Canvas 绘制参数：

- `lineCap = 'round'`
- `lineJoin = 'round'`

### 9.3 图片图层

支持字段：

- `url`
- `x`
- `y`
- `width`
- `height`
- `rotation`
- `opacity`

需要通过现有 `toApiUrl` 或同源代理加载，避免 Canvas 被 CORS 污染。

图片图层带 `rotation` 时，同样使用 `save -> translate(center) -> rotate -> draw -> restore` 的模式，以图层中心点作为旋转中心。绘制完成后必须恢复 `globalAlpha` 和 transform 状态，避免污染后续图层。

### 9.4 贴纸图层

第一版如果贴纸图层内容结构与图片图层一致，可以按图片图层处理。

如果贴纸内容结构未定义或缺少可绘制 URL：

- 跳过该图层
- 记录调试日志
- 不中断导出

### 9.5 调整图层

`adjustment` 图层为预留类型。第一版导出不应用亮度、对比度、滤镜等调整效果。

处理规则：

- 遇到 `adjustment` 图层时跳过
- 记录调试日志
- 后续如果编辑器实际启用 adjustment 图层，再补充 Canvas filter 或像素级处理逻辑

## 10. 图层坐标注意事项

当前文字图层的坐标来自编辑器画布显示尺寸，不一定等于原图像素尺寸。为了保证导出位置准确，建议补充图层坐标基准。

推荐后续在图层 content 中增加：

```ts
canvasWidth: number;
canvasHeight: number;
```

创建或更新图层时写入当前编辑画布尺寸。导出时：

```ts
scaleX = outputWidth / content.canvasWidth;
scaleY = outputHeight / content.canvasHeight;
```

旧数据没有 `canvasWidth/canvasHeight` 时，需要提供兼容逻辑：

- 根据绘本 `aspect_ratio` 推导一个默认编辑画布尺寸
- 或使用当前前端约定的编辑画布基准尺寸

由于同一本绘本页面尺寸一致，这个兼容逻辑可以在单本绘本内保持一致。

## 11. 长图尺寸限制

浏览器 Canvas 对最大宽高有限制。不同浏览器限制不完全一致，建议保守使用：

```ts
const MAX_CANVAS_SIDE = 30000;
```

PNG/JPG 长图高度计算：

```ts
longHeight = pageHeight * pageCount;
```

如果 `longHeight <= MAX_CANVAS_SIDE`：

- 输出单张长图

如果 `longHeight > MAX_CANVAS_SIDE`：

- 自动拆分为多张分卷长图
- 文件名示例：
  - `绘本名_part-1.png`
  - `绘本名_part-2.png`

第一版可以逐个文件下载；更好的体验是后续引入 `jszip`，将分卷打包为 zip。

长图分卷时也要避免一次性保留所有分卷 Canvas。每生成一卷应立即转为 Blob 并释放 Canvas 引用。
