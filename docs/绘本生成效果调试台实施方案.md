# 绘本生成效果调试台实施方案

## 1. 目标

建设一个管理员可见的页面级生成调试台，用于调试正文页图片生成效果。

调试台以绘本为入口。管理员先选择一个绘本，再从该绘本的正文页列表中选择一个页面，查看该页真实生成信息，修改生成变量，预览最终 prompt，并生成调试图片。调试图片写入调试记录，不覆盖正式页面。

本方案的核心不是新增一套调试生成逻辑，而是重构页面生成代码：

```text
真实页面生成入口
调试页面生成入口
        ↓
PageGenerationContext
        ↓
LLMClientBase.generate_page(context)
```

真实生成和调试生成必须使用同一套 `PageGenerationContext`、同一套 prompt 构建逻辑、同一套输入图片组装逻辑、同一个模型调用入口。二者只允许入口、权限、参数来源和写入目标不同。

## 2. 范围

调试对象：

- 正文页图片生成。

调试能力：

- 加载正文页真实生成信息。
- 基于真实生成信息初始化可编辑调试参数。
- 修改页面生成变量。
- 预览最终 prompt。
- 预览模型实际输入资源。
- 调用同一套页面生成代码生成调试图。
- 保存调试图生成记录、调试参数、调试图片 OSS URL、输出图 OSS URL、备注、评分和标签。

调试台不处理封面和封底。

## 3. 核心架构

### 3.1 PageGenerationContext

新增页面生成上下文对象：

```text
app/schemas/page_generation.py
```

定义：

```python
from dataclasses import dataclass, field


@dataclass
class PageGenerationStoryboard:
    summary: str = ""
    visual_brief: str = ""
    anchor_refs: list[str] = field(default_factory=list)
    must_include: list[str] = field(default_factory=list)
    composition: str = ""
    avoid: list[str] = field(default_factory=list)


@dataclass
class PageGenerationVisualAnchor:
    id: str
    type: str
    name: str
    description: str = ""
    key_attributes: list[str] = field(default_factory=list)


@dataclass
class PageGenerationStyleContext:
    name: str = ""
    generation_prompt: str = ""
    negative_prompt: str = ""
    reference_images: list[str] = field(default_factory=list)


@dataclass
class PageGenerationContext:
    cli_type: str
    storybook_id: int | None
    page_id: int | None
    page_index: int

    story_text: str
    storyboard: PageGenerationStoryboard | None
    story_context: list[str]

    visual_anchors: list[PageGenerationVisualAnchor] = field(default_factory=list)

    style: PageGenerationStyleContext = field(default_factory=PageGenerationStyleContext)
    aspect_ratio: str = "16:9"
    image_size: str = "1k"
    image_instruction: str = ""

    previous_page_image: str | None = None
    character_reference_images: list[str] = field(default_factory=list)
    selected_reference_page_images: list[str] = field(default_factory=list)
```

`PageGenerationContext` 是页面图片生成的唯一入参对象。真实生成、页面重生和调试生成都必须先构建它，再调用 LLM client。

字段说明：

| 字段 | 说明 |
|---|---|
| `cli_type` | 当前使用的模型 |
| `storybook_id` | 绘本 ID，调试快照使用 |
| `page_id` | 页面 ID，调试快照使用 |
| `page_index` | 正文页索引，从 0 开始 |
| `story_text` | 当前页文本 |
| `storyboard` | 当前页分镜 |
| `story_context` | 正文页文本上下文 |
| `visual_anchors` | 当前页实际注入的锚点 |
| `style` | 页面生成实际使用的画风文本和参考图 URL |
| `aspect_ratio` | 图片比例 |
| `image_size` | 图片尺寸 |
| `image_instruction` | 本次图片生成指令 |
| `previous_page_image` | 上一正文页图片 |
| `character_reference_images` | 用户上传的角色参考图 |
| `selected_reference_page_images` | 管理员或重生流程选择的页面参考图 |

### 3.2 LLMClientBase 接口改造

修改：

```text
app/services/llm_cli.py
```

把页面生成接口从散参数改为接收 `PageGenerationContext`。

`LLMClientBase` 是模型调用层，不依赖业务 ORM，也不依赖页面、绘本、画风等业务 schema。它不能接收或认识 `ImageStyleVersion`、`Storybook`、`Page`、`Storyboard`、`VisualAnchor` 等业务对象。所有业务对象必须在 `page_generation_service.py` 中解析成 `PageGenerationContext` 内部定义的纯生成 DTO，再传入 client。

目标接口：

```python
class LLMClientBase(ABC):
    @abstractmethod
    def build_page_prompt(self, context: PageGenerationContext) -> str:
        pass

    @abstractmethod
    def build_page_input_resources(
        self,
        context: PageGenerationContext,
    ) -> list[PageGenerationInputResource]:
        pass

    @abstractmethod
    async def generate_page(self, context: PageGenerationContext) -> str:
        pass
```

新增输入资源描述结构：

```python
@dataclass
class PageGenerationInputResource:
    role: str
    url: str
    source: str = ""
    source_id: int | None = None
    sort_order: int = 0
    note: str = ""
```

`role` 取值：

| role | 说明 |
|---|---|
| `style_reference` | 画风参考图 |
| `previous_page` | 上一页连续性图片 |
| `character_reference` | 角色参考图 |
| `selected_page_reference` | 选择的页面参考图 |

`GeminiCli.generate_page(context)` 和 `DoubaoCli.generate_page(context)` 内部必须调用自己的：

```python
prompt = self.build_page_prompt(context)
input_resources = self.build_page_input_resources(context)
```

调试台预览 prompt 时，也调用同一个 `build_page_prompt(context)` 和 `build_page_input_resources(context)`。

```python
async def build_page_generation_context_from_page(

    image_instruction: str = "",
    selected_reference_page_ids: list[int] | None = None,

    ...


async def build_page_generation_context_from_debug_params(
    page_id: int,
    debug_params: PageGenerationDebugParams,

    ...


def build_page_prompt_preview(
    context: PageGenerationContext,
) -> PageGenerationPreview:
    ...


async def generate_page_with_context(
    context: PageGenerationContext,
) -> str:
    ...
```

### 4.1 build_page_generation_context_from_page

从正式数据库构建真实页面生成上下文。

读取：

- `Page`
- `Storybook`
- 正文页列表
- 上一正文页图片
- `Storybook.visual_anchors`
- 当前页 `Page.storyboard.anchor_refs`
- `ImageStyleVersion`，并解析为 `PageGenerationStyleContext`
- `storybook_reference_images` 中的角色参考图 OSS URL
- 选择的页面参考图

输出：

```python
PageGenerationContext
```

### 4.2 build_page_generation_context_from_debug_params

调试生成不复用生产入口生成出来的 `PageGenerationContext`。调试业务根据页面真实数据初始化调试参数表单，管理员提交后，调试业务使用完整的 `PageGenerationDebugParams` 独立组装新的 `PageGenerationContext`。

`PageGenerationDebugParams` 是调试表单 DTO，不是 `PageGenerationContext`，也不是业务 ORM。它保存管理员可编辑的完整调试状态。初始化时由真实页面生成参数转换而来；提交后生成所需变量以该对象为准。

调试参数字段：

| 字段 | 说明 |
|---|---|
| `cli_type` | 模型 |
| `page_index` | 正文页索引 |
| `story_text` | 当前页文本 |
| `storyboard` | 当前页分镜 |
| `story_context` | 正文上下文 |
| `visual_anchors` | 当前页注入锚点 |
| `image_style_id` | 调试时选中的画风 ID，仅用于追溯 |
| `image_style_version_id` | 调试时选中的画风版本 ID，仅用于追溯 |
| `style_name` | 画风名称 |
| `style_generation_prompt` | 画风正向 prompt |
| `style_negative_prompt` | 画风负向 prompt |
| `style_reference_images` | 画风参考图 OSS URL |
| `aspect_ratio` | 图片比例 |
| `image_size` | 图片尺寸 |
| `image_instruction` | 图片指令 |
| `previous_page_image` | 上一页参考图 OSS URL |
| `character_reference_images` | 角色参考图 OSS URL |
| `selected_reference_page_images` | 页面参考图 OSS URL |

### 4.3 build_page_prompt_preview

调用当前模型 client 的公共方法：

```python
client = LLMClientBase.get_client(context.cli_type)
prompt = client.build_page_prompt(context)
resources = client.build_page_input_resources(context)
```

返回：

```python
PageGenerationPreview(
    prompt=prompt,
    input_resources=resources,
)
```

### 4.4 generate_page_with_context

调用：

```python
client = LLMClientBase.get_client(context.cli_type)
return await client.generate_page(context)
```

该函数是正式生成和调试生成的共同入口。

## 5. 真实生成链路改造

### 5.1 首次生成正文页

修改：

```text
app/services/storybook_service.py
```

当前正文页生成从直接调用：

```python
llm_client.generate_page(...)
```

改为：

```python
context = await build_page_generation_context_from_page(page.id)
image_url = await generate_page_with_context(context)
```

正式链路只负责：

- 设置页面状态。
- 调用公共生成服务。
- 上传图片。
- 写回 `Page.image_url`。
- 同步绘本状态。
- 扣费。

正式链路不再手动拼 `generate_page` 的散参数。

### 5.2 单页重新生成

修改：

```text
app/services/page_service.py
```

页面重新生成中的图片生成也改为：

```python
context = await build_page_generation_context_from_page(
    page_id,
    image_instruction=image_instruction,
    selected_reference_page_ids=reference_page_ids,
)
image_url = await generate_page_with_context(context)
```

文本重生和分镜重生逻辑保持原有顺序：

```text
text -> storyboard -> image
```

图片生成阶段不再直接拼散参数。

### 6.2 调试 API

|---|---|---|

| 字段 | 说明 |
|---|---|

| `debug_params` | 根据当前页面真实生成信息转换出的完整 `PageGenerationDebugParams` 初始值 |

### 6.3 调试请求

```python
class GenerationDebugStoryboard(BaseModel):
    summary: str = ""
    visual_brief: str = ""
    anchor_refs: list[str] = []
    must_include: list[str] = []
    composition: str = ""
    avoid: list[str] = []


class GenerationDebugVisualAnchor(BaseModel):
    id: str
    type: str
    name: str
    description: str = ""
    key_attributes: list[str] = []


class PageGenerationDebugParams(BaseModel):
    cli_type: str
    page_index: int

    story_text: str
    storyboard: GenerationDebugStoryboard | None = None
    story_context: list[str]
    visual_anchors: list[GenerationDebugVisualAnchor] = []

    image_style_id: int | None = None
    image_style_version_id: int | None = None
    style_name: str = ""
    style_generation_prompt: str = ""
    style_negative_prompt: str = ""
    style_reference_images: list[str] = []

    aspect_ratio: str
    image_size: str
    image_instruction: str = ""

    previous_page_image: str | None = None
    character_reference_images: list[str] = []
    selected_reference_page_images: list[str] = []
```

调试表单初始化时，`generation_debug_service.py` 可以根据 `image_style_version_id` 读取业务模型，并把画风名称、正向 prompt、负向 prompt 和参考图 OSS URL 填入 `PageGenerationDebugParams`。管理员提交后，生成时只使用 `PageGenerationDebugParams` 中保存的 `style_name`、`style_generation_prompt`、`style_negative_prompt`、`style_reference_images` 组装 `PageGenerationContext.style`，不再从 `ImageStyleVersion` 读取 prompt 内容。

调试接口中的图片字段必须使用 OSS URL。管理员在调试期间上传的角色参考图、画风参考图、上一页替代图或页面参考图，先通过上传接口写入 OSS，再把 OSS URL 写入 `PageGenerationDebugParams`。

`PageGenerationDebugParams` 是调试状态的数据库保存对象。它必须包含管理员最终提交的所有调试参数和图片 OSS URL。prompt、输入资源列表和 `PageGenerationContext` 都由该对象实时生成，不作为数据库字段保存。

Prompt 预览接口只返回预览结果，不写入 `generation_debug_runs`。

Prompt 预览处理流程：

```text
读取 PageGenerationDebugParams
-> 使用 PageGenerationDebugParams 组装调试 PageGenerationContext
-> 调用 build_page_prompt_preview
-> 返回可视化 prompt 页面数据
```

调试图生成处理流程：

```text
读取 PageGenerationDebugParams
-> 使用 PageGenerationDebugParams 组装调试 PageGenerationContext
-> 调用 generate_page_with_context
-> 上传调试生成图到 OSS
-> 保存调试参数和输出图 OSS URL 到 generation_debug_runs
```

## 7. 调试记录模型

新增表：

```text
generation_debug_runs
```

字段：

| 字段 | 说明 |
|---|---|
| `id` | 主键 |
| `storybook_id` | 绘本 ID |
| `page_id` | 页面 ID |
| `admin_user_id` | 管理员 ID |
| `status` | `pending` / `running` / `finished` / `error` |
| `debug_params` | 管理员提交的 `PageGenerationDebugParams`，包含所有调试变量和图片 OSS URL |
| `output_image_url` | 调试生成图 OSS URL |
| `error_message` | 错误信息 |
| `rating` | 评分 |
| `tags` | 标签 |
| `notes` | 备注 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

调试图不写入正式 `Page.image_url`。

`generation_debug_runs` 不保存最终 prompt，不保存输入资源快照，不保存 `PageGenerationContext` 快照。最终 prompt 不是单纯文本，它和输入图片共同构成模型请求，应通过 `debug_params` 实时生成可视化预览。

数据库只保存两类内容：

- 管理员调试参数：`debug_params`
- 调试结果：`output_image_url`、`status`、`error_message`、`rating`、`tags`、`notes`

所有参与调试的图片都必须是 OSS URL：

- 用户创建绘本时上传的角色参考图。
- 管理员调试期间上传的画风参考图。
- 管理员调试期间上传的角色参考图。
- 管理员调试期间上传的上一页替代图。
- 管理员调试期间上传的页面参考图。
- 调试生成输出图。

## 8. 角色参考图持久化

新增：

```text
storybook_reference_images
```

字段：

| 字段 | 说明 |
|---|---|
| `id` | 主键 |
| `storybook_id` | 绘本 ID |
| `image_url` | 参考图 URL |
| `reference_type` | `character` |
| `sort_order` | 顺序 |
| `created_at` | 创建时间 |

创建绘本时，用户上传的角色参考图必须先上传到 OSS，再保存为 `storybook_reference_images`。后续真实生成、单页重生和调试台都从该表读取角色参考图 OSS URL。

`PageGenerationContext.character_reference_images` 来自该表。

创建绘本同步阶段处理顺序：

```text
接收 req.images
-> 上传每张角色参考图到 OSS
-> 写入 storybook_reference_images
-> 构建 PageGenerationContext 时从 storybook_reference_images 读取
-> 正文页生成使用持久化后的 OSS URL
```

后台生成任务不再依赖临时的 `req.images` base64 参数。真实生成和调试还原都以数据库中保存的 `storybook_reference_images` 为准。

## 12. 验收标准

- `LLMClientBase.generate_page` 只接收 `PageGenerationContext`。
- `GeminiCli.generate_page` 和 `DoubaoCli.generate_page` 都接收 `PageGenerationContext`。
- 真实首次正文页生成通过 `generate_page_with_context`。
- 单页重新生成通过 `generate_page_with_context`。
- 调试图生成通过 `generate_page_with_context`。
- Prompt 预览调用同一个 client 的 `build_page_prompt(context)`。
- 输入资源预览调用同一个 client 的 `build_page_input_resources(context)`。
- 调试图不覆盖正式页面。
- 调试 API 只有管理员可访问。
- 角色参考图持久化，并进入 `PageGenerationContext.character_reference_images`。
- 创建绘本时 `visual_anchors` 能保存到 `Storybook.visual_anchors`。
