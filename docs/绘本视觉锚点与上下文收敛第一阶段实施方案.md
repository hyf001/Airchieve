# 绘本视觉锚点与上下文收敛第一阶段实施方案

## 1. 阶段目标

第一阶段只做“上下文收敛”，目标是减少正文页图片生成时的注意力干扰，让模型优先画好当前页，并稳定遵守锁定画风。

本阶段解决的问题：

- 图片生成 prompt 默认包含完整故事全文，导致当前页分镜权重被稀释。
- 图片生成 prompt 直接使用当前页故事原文，原文里可能有心理描写、对话或抽象叙事，不够聚焦画面目标。
- 分镜生成阶段默认产出 `color`、`lighting`，导致图片生成阶段天然带入额外色彩光影目标。
- 连续性是绘本生成的最高优先级，但当前 prompt 没有把连续性规则说清楚，容易在完整故事、画风、分镜和上一页图片之间摇摆。
- 画风描述、完整故事、分镜色彩光影、上一页图片同时竞争注意力，导致画风不稳定。

本阶段不解决的问题：

- 不新增视觉锚点数据模型。
- 不迁移到弱结构化分镜。
- 不做生成后 QA 检查。
- 不做自动修复闭环。
- 不新增锚点参考图沉淀逻辑。

## 2. 当前链路现状

### 2.1 Gemini

当前入口：

```text
app/services/gemini_cli.py
GeminiCli.generate_page
```

当前行为：

- 接收 `story_context`，并拼成 `Complete Story Context`。
- 分镜生成和分镜重生成 prompt 要求返回 `scene`、`characters`、`shot`、`color`、`lighting`。
- 图片生成 prompt 使用当前页故事原文 `story_text`。
- 图片 prompt 中明确要求模型参考完整故事上下文。
- 上一页图片说明为 established rendering，这是正确方向，但需要把连续性提升为明确最高优先级。
- 画风参考图目前会作为每页参考图传入，容易和上一页连续性、角色参考图竞争注意力。

主要风险：

```text
完整故事上下文容易干扰当前页，但上一页连续性参考必须保留为最高优先级
```

### 2.2 豆包

当前入口：

```text
app/services/doubao_cli.py
_build_image_prompt
DoubaoCli.generate_page
```

当前行为：

- 结构化分镜 schema `_StoryboardDetail` 要求 `color`、`lighting`。
- 分镜生成和分镜重生成 prompt 要求返回 `scene`、`characters`、`shot`、`color`、`lighting`。
- 图片生成 prompt 使用当前页故事原文 `story_text`。
- `_build_image_prompt` 会拼接完整故事上下文。
- prompt 要求“为第 N 页生成插图”，但仍把完整故事放在前面。
- 上一页图片说明要求保持角色形象、艺术风格、色调和环境连贯，这是正确方向，但需要把连续性写成最高优先级。
- 参考图说明目前容易混用“画风参考”和“角色参考”语义。

主要风险：

```text
完整故事上下文容易干扰当前页；上一页图应作为最高优先级连续性参考保留
```

## 3. 本阶段改造原则

### 3.1 不再默认传完整故事全文

正文页图片生成只需要当前页信息：

- 当前页视觉摘要。
- 当前页分镜。
- 锁定画风。
- 用户图片调整指令。
- 必要的上一页图片连续性参考。

`story_context` 参数可以保留，避免大范围改接口，但图片 prompt 不再默认展开完整故事。

### 3.2 当前页优先

图片生成 prompt 的语义顺序应改为：

```text
当前页任务
当前页视觉摘要
当前页分镜
锁定画风
硬性规则
参考图说明
禁止项
```

不再把“理解完整故事”作为图片生成目标。

### 3.3 连续性最高优先级

绘本是连续叙事，跨页连续性必须是图片生成的最高优先级。

连续性包含：

- 同一角色的外观、服装、配饰、比例、气质保持一致。
- 同一关键物的颜色、材质、形状、标志性细节保持一致。
- 同一核心场景的空间关系和主要视觉特征保持一致。
- 同一本书的整体画风、媒介质感、线条、色彩体系保持一致。

上一页图片是当前阶段最重要的连续性参考。不能因为去掉完整故事上下文而削弱上一页图片的作用。

明确优先级：

```text
1. 跨页视觉连续性
2. 锁定画风
3. 当前页视觉摘要和分镜
4. 用户明确的图片调整指令
```

如果上一页图片和锁定画风存在轻微差异，优先保持整本书已经建立的连续外观。只有当用户明确要求更换画风或重新生成整本风格时，才允许打破已经建立的连续性。

### 3.4 分镜生成不再产出色彩光影字段

新生成和重新生成分镜时，只允许产出：

```json
{
  "scene": "...",
  "characters": "...",
  "shot": "...",
  "summary": "..."
}
```

不再要求模型生成：

```json
{
  "color": "...",
  "lighting": "..."
}
```

兼容已有旧分镜数据时，图片生成 prompt 也不再注入 `color`、`lighting`：

- 读取新分镜时使用 `summary`、`scene`、`characters`、`shot`。
- 读取旧分镜时如果没有 `summary`，可以临时使用当前页故事原文作为兼容兜底，但新生成和重生成的数据必须有 `summary`。
- `color`、`lighting` 即使存在也直接忽略。
- 后续如果某页确实需要特殊色彩或光影，应写入 `scene` 或用户明确的 `image_instruction`，不要再通过固定字段默认传入。

这样可以避免每页自动生成的色调、光影字段与锁定画风竞争注意力。

### 3.5 每页生成视觉摘要

分镜生成阶段必须为每一页生成 `summary`。

`summary` 是给图片生成模型使用的当前页视觉摘要，不是给用户阅读的正文。正文仍然保存在 `text` 字段中，用于页面展示、编辑和导出。

落库位置：

```text
Page.text = 绘本正文原文
Page.storyboard.summary = 当前页视觉摘要
Page.storyboard.scene / characters / shot = 当前页分镜
```

这样不需要修改 `generate_page` 的函数签名，图片生成时可以从 `storyboard["summary"]` 读取摘要。

摘要要求：

- 一句话说明当前页要画什么。
- 只保留可视化信息：角色、动作、关键物、场景、情绪。
- 不写心理活动、旁白解释、抽象主题、长对话。
- 不引入原文没有的剧情事实。
- 中文建议 30-60 字，英文建议 20-40 词。

示例：

```json
{
  "text": "小熊想起妈妈说过的话，心里慢慢有了勇气。他抱着红色杯子，走向洒满阳光的窗边。",
  "storyboard": {
    "summary": "小熊抱着红色杯子，走向阳光照进来的窗边，表情逐渐坚定。",
    "scene": "sunlit room with a bright window",
    "characters": "little bear walking toward the window, holding a cup with a determined expression",
    "shot": "medium shot, bear centered, window light in the background"
  }
}
```

图片生成时使用 `summary`，不再直接使用 `text`。

### 4.5 参考图策略调整

本阶段不新增数据库层面的参考图类型，只调整图片生成时的参考图用途和传入时机。

图片生成阶段输入给大模型的图片只分三类：

| 类型 | 含义 | 使用时机 |
|---|---|---|
| 风格参考图片 | 风格主图片，只用于建立整本绘本的锁定画风 | 只在创建第一页时传入 |
| 连续性参考图片 | 上一页图片，用于保持跨页视觉连续性 | 第二页及以后传入 |
| 角色参考图片 | 用户上传或系统选取的角色参考图，角色不一定是人，也可以是动物、玩偶等主要角色 | 第一页及后续页面都可传入 |

推荐策略：

```text
第一页：
1. 风格参考图片
2. 角色参考图片

第二页及以后：
1. 连续性参考图片
2. 角色参考图片
```

禁止组合：

```text
第一页不传连续性参考图片，因为第一页没有上一页。
第2到N页不传风格参考图片，因为画风应由上一页连续性图片延续。
角色参考图片两种场景都可以传。
```

原因：

- 风格主图片只用于第一页定调，避免每页重复上传画风图造成注意力竞争和额外成本。
- 第二页及以后通过上一页图片延续已经建立的画风和角色外观。
- 现有引用参考图片明确为角色参考图片，只用于保持角色身份和外观可辨识，不承担画风参考职责。
- 上一页图片是连续性参考图片，也是最高优先级连续性参考。

Gemini 使用 multipart parts 时，也建议按这个顺序添加说明和图片。

豆包使用 `image` 参数时，也建议按这个顺序构建 `input_images`，并同步更新 `【参考图片说明】` 中的序号。

风格主图片来源：

```text
优先使用 image_style_version.reference_images 中 is_cover=true 的图片。
如果没有 is_cover=true，则使用排序后的第一张画风参考图。
```

其他画风参考图本阶段不传入图片生成 prompt。

角色参考图片说明：

```text
现有的引用参考图片、用户上传参考图片、页面重生成时选择的参考页面图片，在图片生成 prompt 中统一标注为角色参考图片。
角色不一定是人，也可以是动物、玩偶、拟人化物体等主要角色。
它们只用于保持角色身份、外观、姿态特征和可辨识性。
不要从角色参考图片中学习画风、色调、背景复杂度或构图。
```

## 5. 接口兼容策略

### 5.1 不改 LLMClientBase 签名

本阶段不改：

```text
app/services/llm_cli.py
generate_page(...)
```

`story_context` 仍然保留，调用方不用改。

### 5.2 不改 storybook_service 调用

本阶段不要求修改：

```text

app/services/page_service.py
```

这些服务仍可传入完整 `story_context`，但 Gemini / 豆包客户端内部不再默认把它放进图片 prompt。

### 5.3 更新 Storyboard 类型

本阶段需要更新 `Storyboard` 结构定义，但不新增数据库字段。

修改文件：

```text
app/models/page.py
app/schemas/page.py
```

目标结构：

```python
class Storyboard(TypedDict):
    summary: str     # 当前页视觉摘要，用于图片生成
    scene: str       # 场景环境
    characters: str  # 人物动作、姿态、表情
    shot: str        # 景别构图
```

移除：

```python
color: str
lighting: str
```

数据库层仍使用原有 JSON 字段 `Page.storyboard`，不需要新增迁移。旧数据里如果存在 `color`、`lighting`，读取时忽略即可；旧数据如果没有 `summary`，图片生成可临时使用 `Page.text` 兜底。

### 5.4 可选开关

如果担心上线后效果波动，可以增加配置开关：

```text
IMAGE_PROMPT_CONTEXT_MODE=current_page_only
```

可选值：

| 值 | 行为 |
|---|---|
| `current_page_only` | 默认新逻辑，不注入完整故事 |
| `full_story` | 回退旧逻辑，注入完整故事 |

首版也可以不加开关，直接改为新逻辑。若线上生成质量需要快速回退，再通过代码回滚。

- Gemini 正文页图片生成成功。
- 豆包正文页图片生成成功。

- 第一页生成时，风格主图片作为唯一画风图片传入并在 prompt 中说明。
- 第二页及以后不再传入任何画风图片。
- 上一页图片仍可传入，且 prompt 明确它是最高优先级连续性参考。
- 页面重新生成图片链路仍可用。

### 6.2 质量验收

用同一批故事对比旧逻辑和新逻辑：

- 当前页主体是否更贴近分镜。
- 图片是否更贴近当前页摘要。
- 锁定画风是否更稳定。
- 是否减少“第一页风格偏差被后续持续放大”的情况。
- 是否减少模型把完整故事里的其他页面元素画进当前页。
- 是否没有明显降低角色连续性。

建议至少验证：

```text
3 本故事
每本 6-10 页
每本至少 1 个重复角色
每本至少 1 个锁定图片风格
Gemini 和豆包各跑一轮
```

### 6.3 成本验收

本阶段理论上不增加模型调用次数。

预期变化：

| 项 | 变化 |
|---|---|
| 图片生成调用次数 | 不变 |
| 图片生成耗时 | 基本不变，可能略降 |
| 文本 token | 下降 |
| 图片输入数量 | 不变 |
| 生成失败率 | 不应上升 |

如果一页原来注入完整 10 页故事，新逻辑会减少约数百到数千文本 token，具体取决于每页故事长度。

## 7. 测试建议

### 7.1 单元测试

如果现有项目没有 prompt 构建测试，可以补轻量测试：

- Gemini prompt 不包含 `Complete Story Context`。
- Gemini prompt 包含当前页 `summary`。
- Gemini prompt 不直接包含当前页故事原文。
- Gemini prompt 包含锁定画风字段。
- Gemini 第一页 prompt 包含风格主图片说明。
- Gemini 第二页及以后 prompt 不包含画风图片说明。
- Gemini 新分镜结果不包含 `color`、`lighting`。
- 豆包 prompt 不包含 `完整故事上下文`。
- 豆包 prompt 包含当前页 `summary`。
- 豆包 prompt 不直接包含当前页故事原文。
- 豆包 prompt 包含优先级规则。
- 豆包第一页 prompt 包含风格主图片说明。
- 豆包第二页及以后 prompt 不包含画风图片说明。
- 豆包新分镜结果不包含 `color`、`lighting`。

### 7.2 手工测试

建议手工跑以下场景：

1. 无画风版本，仅模板风格。
2. 有画风版本和风格主图片。
3. 有角色参考图片。
4. 有上一页图片。
5. 页面重新生成时带 `image_instruction`。
6. 分镜中有明显 `color` / `lighting` 字段，但图片 prompt 不应包含这两个字段。

重点观察：

- 当前页有没有被其他页文本污染。
- 上一页图是否仍能提供基本连续性。
- 锁定画风是否比旧逻辑更强。

## 8. 实施步骤

推荐按以下顺序实施：

1. 修改 Gemini 分镜生成和分镜重生成 prompt，移除 `color`、`lighting`。
2. 修改 Gemini 分镜生成和分镜重生成 prompt，新增每页 `summary`。
3. 修改豆包分镜 schema、分镜生成 prompt、分镜重生成 prompt，移除 `color`、`lighting`。
4. 修改豆包分镜 schema、分镜生成 prompt、分镜重生成 prompt，新增每页 `summary`。
5. 修改 Gemini `generate_page` 的 system instruction 和 prompt，使用 `summary` 替代当前页故事原文。
6. 修改 Gemini 参考图说明和传入逻辑：第一页传风格参考图片，第二页及以后传连续性参考图片，现有引用参考图片标注为角色参考图片。
7. 修改豆包 `_build_image_prompt`，移除完整故事上下文，并使用 `summary` 替代当前页故事原文。
8. 修改豆包参考图收集顺序和说明：第一页传风格参考图片，第二页及以后传连续性参考图片，现有引用参考图片标注为角色参考图片。
9. 增加必要日志。
10. 本地跑 lint 或后端测试。
11. 手工生成 1 本短绘本，确认链路可用。
12. 对比旧样例，记录画风稳定性和当前页准确性。

## 9. 风险与回退

### 9.1 角色连续性可能变弱

移除完整故事上下文后，模型对全书叙事的理解会减少。

缓解：

- 保留上一页图片作为最高优先级连续性参考。
- 保留角色参考图片。
- 第二阶段再通过 `anchor_refs` 和视觉锚点增强连续性。

### 9.2 某些跨页伏笔表达变弱

如果当前页需要理解全书伏笔，移除完整故事可能让模型少画某些暗示。

缓解：

- 第一阶段接受这个取舍。
- 后续可以增加极短 `narrative_note`，但不恢复完整故事全文。

### 9.3 旧 prompt 效果在个别故事上更好

个别故事可能依赖完整上下文。

回退：

- 如果增加配置开关，可以把 `IMAGE_PROMPT_CONTEXT_MODE` 改回 `full_story`。
- 如果没有开关，回滚 Gemini / 豆包 prompt 改动。

## 10. 完成定义

第一阶段完成时，应满足：

- Gemini 和豆包正文页图片 prompt 默认不再包含完整故事全文。
- Gemini 和豆包新生成/重生成的分镜默认不再包含 `color`、`lighting`。
- Gemini 和豆包新生成/重生成的分镜每页都包含 `summary`。
- 当前页摘要、当前页分镜、锁定画风成为图片 prompt 主体。
- 图片生成 prompt 使用 `summary`，不再直接使用当前页故事原文。
- 风格主图片只在第一页作为唯一画风图片进入图片生成 prompt。
- 现有引用参考图片明确为角色参考图片。
- 上一页图片在文案上明确为最高优先级连续性参考。
- 参考图用途说明更清晰，既保留锁定画风，也不削弱跨页连续性。
- 不引入新的数据库结构、QA 调用或自动修复流程。
