# 绘本生成流程与阶段参数梳理

## 1. 总览

当前绘本生成是一个前端向导 + 后端异步生成流程。

主流程：

```text
输入指令
-> 生成/填写故事
-> 生成分镜和视觉锚点
-> 用户确认/编辑分镜
-> 创建绘本记录和页面占位
-> 后台逐页生成正文图片
-> 后台生成封面图片
-> 封底使用固定图片
-> 绘本完成
```

对应前端页面：

```text
frontend/src/pages/HomeView.tsx
```

对应主要后端入口：

```text
POST /api/v1/storybooks/story
POST /api/v1/storybooks/storyboard
POST /api/v1/storybooks/from-story
GET  /api/v1/storybooks/{storybook_id}/status
GET  /api/v1/storybooks/{storybook_id}
```

## 2. 阶段一：输入指令与故事参数

### 2.1 前端来源

前端在首页创建向导的输入阶段收集：

| 参数 | 来源 | 说明 |
|---|---|---|
| `prompt` | 用户输入 | 故事主题、正文或创作指令 |
| `mode` | 前端状态 | `ai` 表示 AI 生成故事，`manual` 表示用户直接输入故事正文 |
| `word_count` | `storyParams.word_count` | 目标字数，默认 500 |
| `story_type` | `storyParams.story_type` | 故事类型 |
| `language` | `storyParams.language` | 语言 |
| `age_group` | `storyParams.age_group` | 年龄段 |
| `cli_type` | `creationParams.cli_type` | 生成模型，默认 `doubao` |

### 2.2 AI 生成故事接口

当 `mode = ai` 时，前端调用：

```text
POST /api/v1/storybooks/story
```

请求模型：

```python
CreateStoryRequest
```

请求参数：

| 字段 | 类型 | 说明 |
|---|---|---|
| `instruction` | `str` | 用户指令/故事描述 |
| `word_count` | `int` | 目标字数，100-2000 |
| `story_type` | `StoryType` | 故事类型 |
| `language` | `Language` | 语言 |
| `age_group` | `AgeGroup` | 年龄组 |
| `cli_type` | `CliType` | 模型：`gemini` / `doubao` |

后端调用：

```text
storybook_service.create_story_only
-> LLMClientBase.get_client(cli_type)
-> llm_client.create_story(...)
```

输出：

```python
CreateStoryResponse
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `title` | `str` | 故事标题 |
| `content` | `str` | 故事正文 |

### 2.3 手动输入故事

当 `mode = manual` 时，不调用后端故事生成接口。

前端直接设置：

| 字段 | 值 |
|---|---|
| `storyTitle` | `prompt.slice(0, 10)` |
| `storyContent` | `prompt` |
| `originalInstruction` | `prompt` |

然后进入故事预览阶段。

## 3. 阶段二：故事确认与分镜生成

### 3.1 前端输入

用户在故事预览阶段确认：

| 参数 | 来源 | 说明 |
|---|---|---|
| `title` | 用户确认/编辑后的标题 | 绘本标题 |
| `content` | 用户确认/编辑后的故事 | 故事正文 |
| `pageCount` | 用户选择 | 正文页数 |
| `selectedImageStyle.id` | 当前选中画风 | 图片风格 ID |
| `creationParams.cli_type` | 创建参数 | 分镜生成模型 |

当前代码调用：

```text
generateStoryboard({
  title,
  story_content: content,
  page_count: pageCount,
  cli_type: creationParams.cli_type,
  image_style_id: selectedImageStyle.id
})
```

注意：后端 schema 已支持 `has_character_reference_images`，但当前前端 service 类型尚未暴露该字段。

### 3.2 分镜生成接口

接口：

```text
POST /api/v1/storybooks/storyboard
```

请求模型：

```python
GenerateStoryboardRequest
```

请求参数：

| 字段 | 类型 | 说明 |
|---|---|---|
| `title` | `Optional[str]` | 绘本标题，用于生成封面分镜 |
| `story_content` | `str` | 故事正文 |
| `page_count` | `int` | 正文页数，1-20 |
| `cli_type` | `CliType` | 模型 |
| `image_style_id` | `int` | 图片风格 ID |
| `has_character_reference_images` | `bool` | 是否存在角色参考图 |

后端处理：

```text
storybook_service.create_storyboard_only
-> validate_style_available(image_style_id)
-> 获取当前发布的 ImageStyleVersion
-> 收集 style_version.reference_images
-> llm_client.create_storyboard_from_story(...)
```

传给 LLM 的关键参数：

| 参数 | 来源 | 说明 |
|---|---|---|
| `story_content` | 请求 | 故事正文 |
| `page_count` | 请求 | 正文页数 |
| `style_name` | `ImageStyle.name` | 画风名称 |
| `style_reference_images` | `ImageStyleVersion.reference_images` | 画风参考图 URL |
| `has_character_reference_images` | 请求 | 用于决定是否生成 `character` 锚点 |

### 3.3 分镜生成输出

后端返回：

```python
GenerateStoryboardResponse
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `storyboards` | `list[StoryboardItemResponse]` | 页面文本和分镜列表，包含封面页 |
| `visual_anchors` | `list[VisualAnchor]` | 绘本级轻量视觉锚点 |

`storyboards` 的第一项是后端额外构造的封面页：

| 字段 | 值 |
|---|---|
| `text` | `req.title or "封面"` |
| `page_type` | `cover` |
| `storyboard` | `build_cover_storyboard(...)` |

后续项是正文页：

| 字段 | 来源 |
|---|---|
| `text` | LLM 拆分后的每页正文 |
| `storyboard` | LLM 生成并 normalize 后的分镜 |
| `page_type` | `content` |

### 3.4 Storyboard 结构

当前阶段二弱分镜结构：

| 字段 | 类型 | 说明 |
|---|---|---|
| `summary` | `str` | 当前页视觉摘要 |
| `visual_brief` | `str` | 自然语言画面目标 |
| `anchor_refs` | `list[str]` | 当前页引用的锚点 ID |
| `must_include` | `list[str]` | 本页必须出现的少量元素 |
| `composition` | `str` | 构图要求 |
| `avoid` | `list[str]` | 禁止项 |

### 3.5 VisualAnchor 结构

```python
VisualAnchor
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `str` | 锚点 ID |
| `type` | `str` | `character` / `object` |
| `name` | `str` | 名称 |
| `description` | `str` | 描述 |
| `key_attributes` | `list[str]` | 关键属性，可选 |

后端会执行：

```text
normalize_visual_anchors(...)
clean_storyboards_anchor_refs(...)
```

规则要点：

- 有角色参考图时过滤 `character` 锚点。
- 不接受 `scene` 锚点。
- `character` 最多 3 个。
- `object` 最多 5 个。
- 每页 `anchor_refs` 最多 3 个。
- 非法锚点引用会被清理。

## 4. 阶段三：分镜确认与编辑

### 4.1 前端输入

用户在分镜编辑页可以编辑：

| 内容 | 数据 |
|---|---|
| 页面文本 | `StoryboardItem.text` |
| 页面分镜 | `StoryboardItem.storyboard` |
| 页面顺序 | `storyboards` 数组顺序 |
| 删除页面 | 从 `storyboards` 中移除 |

确认后前端保存：

```text
setStoryboards(updatedStoryboards)
setStep("creating")
```

### 4.2 输出

输出是即将创建绘本的页面数组：

```typescript
pages = storyboards.map((item) => ({
  text: item.text,
  image_url: "",
  storyboard: item.storyboard,
  page_type: item.page_type || "content",
}))
```

## 5. 阶段四：创建绘本记录与页面占位

### 5.1 前端输入

创建绘本阶段前端传入：

| 参数 | 来源 | 说明 |
|---|---|---|
| `title` | `storyTitle` | 绘本标题 |
| `description` | `originalInstruction.slice(0, 200)` | 用户原始输入摘要 |
| `image_style_id` | 创建页选择 | 图片风格 ID |
| `images` | 创建页上传 | 角色参考图 base64 列表 |
| `cli_type` | 创建参数 | 生成模型 |
| `aspect_ratio` | 创建参数 | 图片比例 |
| `image_size` | 创建参数 | 图片尺寸 |
| `pages` | 分镜确认结果 | 页面文本和分镜 |

当前前端 `CreateStorybookFromStoryRequest` 类型尚未包含 `visual_anchors`，service 请求体也没有传 `visual_anchors`。后端已支持该字段。

### 5.2 创建绘本接口

接口：

```text
POST /api/v1/storybooks/from-story
```

请求模型：

```python
CreateStorybookFromStoryRequest
```

请求参数：

| 字段 | 类型 | 说明 |
|---|---|---|
| `title` | `str` | 绘本标题 |
| `description` | `str` | 绘本描述/用户原始输入 |
| `template_id` | `Optional[int]` | 已废弃，传入会报错 |
| `image_style_id` | `Optional[int]` | 图片风格 ID |
| `images` | `Optional[list[str]]` | 角色参考图 base64 列表 |
| `visual_anchors` | `Optional[list[VisualAnchor]]` | 绘本级视觉锚点 |
| `cli_type` | `CliType` | 生成模型 |
| `aspect_ratio` | `AspectRatio` | 图片比例 |
| `image_size` | `ImageSize` | 图片尺寸 |
| `pages` | `list[StorybookPage]` | 页面输入 |

`StorybookPage`：

| 字段 | 类型 | 说明 |
|---|---|---|
| `text` | `str` | 页面文字 |
| `image_url` | `str` | 创建时为空 |
| `storyboard` | `Optional[Storyboard]` | 页面分镜 |
| `page_type` | `str` | `cover` / `content` |

### 5.3 同步阶段后端处理

接口先调用：

```text
storybook_service.create_storybook_from_story_async(...)
```

同步阶段做这些事：

1. normalize `visual_anchors`。
2. 根据 `has_character_reference_images=bool(req.images)` 再次过滤不合法锚点。
3. 检查积分。
4. 校验图片风格可用，并锁定当前发布的画风版本。
5. 创建 `Storybook`。
6. 创建封面页占位。
7. 创建正文页占位。
8. 创建封底页，使用固定封底图。
9. 提交数据库事务。

### 5.4 Storybook 落库字段

| 字段 | 来源 | 说明 |
|---|---|---|
| `title` | 请求 | 绘本标题 |
| `description` | 请求 | 绘本描述 |
| `creator` | 当前用户 ID | 字符串形式 |
| `instruction` | 请求 description | 用户输入摘要 |
| `template_id` | 固定 `None` | 旧模板废弃 |
| `image_style_id` | 请求 | 图片风格 ID |
| `image_style_version_id` | 当前发布版本 | 创建时锁定 |
| `visual_anchors` | 请求后 normalize | 绘本级锚点 |
| `cli_type` | 请求 | 模型 |
| `aspect_ratio` | 请求 | 图片比例 |
| `image_size` | 请求 | 图片尺寸 |
| `status` | `init` | 初始状态 |

### 5.5 Page 落库规则

同步阶段会生成完整页面占位：

```text
封面 page_index=0
正文 page_index=1..n
封底 page_index=n+1
```

封面页：

| 字段 | 说明 |
|---|---|
| `text` | 请求中的封面页文本，缺省用标题 |
| `image_url` | 空字符串 |
| `storyboard` | 封面分镜，清理非法锚点引用 |
| `page_type` | `cover` |
| `status` | `pending` |

正文页：

| 字段 | 说明 |
|---|---|
| `text` | 页面正文 |
| `image_url` | 空字符串 |
| `storyboard` | 页面分镜，清理非法锚点引用 |
| `page_type` | `content` |
| `status` | `pending` |

封底页：

| 字段 | 说明 |
|---|---|
| `text` | 空 |
| `image_url` | 固定封底图 URL |
| `storyboard` | 空 |
| `page_type` | `back_cover` |
| `status` | `finished` |

### 5.6 同步接口输出

接口立即返回：

```python
StorybookCreateResponse
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `int` | 绘本 ID |
| `title` | `str` | 绘本标题 |
| `status` | `str` | 固定返回 `init` |
| `image_style_version_id` | `Optional[int]` | 锁定的画风版本 ID |

同时启动后台任务：

```text
run_create_storybook_from_story_background(
  storybook_id,
  current_user.id,
  req.aspect_ratio.value,
  req.image_size.value,
  req.cli_type,
  req.images
)
```

## 6. 阶段五：后台生成正文图片

### 6.1 后台任务输入

```python
run_create_storybook_from_story_background
```

参数：

| 字段 | 类型 | 说明 |
|---|---|---|
| `storybook_id` | `int` | 绘本 ID |
| `user_id` | `int` | 用户 ID |
| `aspect_ratio` | `str` | 图片比例 |
| `image_size` | `str` | 图片尺寸 |
| `cli_type` | `CliType` | 生成模型 |
| `images` | `Optional[list[str]]` | 角色参考图 base64 列表 |

### 6.2 初始化处理

后台任务会：

1. 读取绘本详情。
2. 将绘本状态改为 `creating`。
3. 分离页面类型：`cover`、`content`。
4. 获取锁定的 `image_style_version`。
5. 构造正文文本上下文 `story_texts`。

若没有页面、没有正文页、或没有锁定画风版本，会进入错误状态。

### 6.3 每个正文页的输入

对每个正文页，调用：

```python
llm_client.generate_page(...)
```

输入参数：

| 参数 | 来源 | 说明 |
|---|---|---|
| `story_text` | 当前 `Page.text` | 当前页故事文本 |
| `storyboard` | 当前 `Page.storyboard` | 当前页分镜 |
| `story_context` | 所有正文页文本 | 全书正文上下文 |
| `page_index` | 正文页索引，从 0 开始 | 不含封面 |
| `character_reference_images` | 后台任务 `images` | 用户上传角色参考图 |
| `previous_page_image` | 前一正文页 `image_url` | 上一页连续性参考 |
| `image_style_version` | 锁定画风版本 | 画风 prompt 和参考图 |
| `aspect_ratio` | 创建参数 | 图片比例 |
| `image_size` | 创建参数 | 图片尺寸 |
| `visual_anchors` | `anchors_for_storyboard(...)` | 当前页引用的锚点 |

`previous_page_image` 规则：

```text
第一页正文：None
第 i 页正文：content_pages[i - 1].image_url
```

`visual_anchors` 规则：

```text
读取 Page.storyboard.anchor_refs
-> 从 Storybook.visual_anchors 中解析对应锚点
-> 最多注入 3 个
```

### 6.4 每个正文页的输出

LLM 返回：

| 输出 | 说明 |
|---|---|
| `image_url` | 通常为 base64 data URL |

后端随后：

1. 设置 `page.image_url = image_url`。
2. 设置 `page.status = finished`。
3. 清空 `page.error_message`。
4. 上传页面图到 OSS。
5. 将更新后的页面 merge 回数据库。

## 7. 阶段六：后台生成封面图片

正文页全部完成后，后台生成封面。

### 7.1 封面参考图选择

后端刷新绘本页面后调用：

```text
page_service.pick_cover_reference_pages(...)
```

从已生成正文页中选取若干参考页。

### 7.2 封面图片输入

调用：

```python
llm_client.generate_cover(...)
```

输入参数：

| 参数 | 来源 | 说明 |
|---|---|---|
| `title` | `Storybook.title` | 绘本标题 |
| `cover_text` | `build_cover_description(title, cover_page)` | 封面文字/描述 |
| `reference_images` | 正文参考页图片 | 从正文页自动选取 |
| `aspect_ratio` | 创建参数 | 图片比例 |
| `image_size` | 创建参数 | 图片尺寸 |
| `image_style_version` | 锁定画风版本 | 画风 prompt 和参考图 |

### 7.3 封面图片输出

LLM 返回封面图 URL 后，后端：

1. 设置 `cover_page.image_url`。
2. 设置 `cover_page.status = finished`。
3. 上传封面图到 OSS。
4. merge 回数据库。

## 8. 阶段七：完成、扣费与错误处理

### 8.1 成功完成

当正文和封面均成功后：

| 对象 | 状态 |
|---|---|
| 正文页 | `finished` |
| 封面页 | `finished` |
| 封底页 | `finished` |
| 绘本 | `finished` |

随后扣除积分：

```text
正文页数 + 1（封面）
```

### 8.2 错误处理

如果 LLM 抛出 `LLMError`：

- 当前活跃页面标记为 `error`。
- `page.error_message = e.user_message`。
- 根据页面状态同步绘本状态。

如果出现其他异常：

- 当前活跃页面标记为 `error`。
- `page.error_message = str(e)[:500]`。
- 根据页面状态同步绘本状态。

已生成成功的页面会保留，用户后续可逐页重新生成。

### 8.3 中止处理

后台每生成一页前会检查：

```text
_is_terminated(storybook_id)
```

如果已中止，则标记绘本为 `terminated` 并停止后续生成。

## 10. 单页重新生成流程

用户在编辑器中可对单页重新生成。

接口：

```text
POST /api/v1/pages/{page_id}/regenerate
```

请求：

```python
RegeneratePageRequest
```

| 字段 | 说明 |
|---|---|
| `regenerate_text` | 是否重写页面文本 |
| `text_instruction` | 文本重写指令 |
| `regenerate_storyboard` | 是否重写分镜 |
| `storyboard_instruction` | 分镜重写指令 |
| `regenerate_image` | 是否重生图片 |
| `image_instruction` | 图片调整指令 |
| `reference_page_ids` | 用户选择的参考页 ID |

同步阶段：

- 校验页面存在。
- 封底页不允许 AI 重生。
- 校验当前用户是绘本创建者。
- 绘本不能处于 `creating` / `updating`。
- 若重生图片，检查积分。
- 绘本状态设为 `updating`。
- 页面状态设为 `generating`。

后台阶段执行顺序：

```text
text -> storyboard -> image
```

重生图片时调用 `llm_client.generate_page(...)`，输入与首生成相似，但有两点差异：

- `character_reference_images` 来自 `reference_page_ids` 选中的正文页图片，而不是创建时上传的角色参考图。
- `image_instruction` 来自本次重生请求。

重生完成后：

- 页面图上传 OSS。
- 页面状态变 `finished`。
- 同步绘本状态。
- 消耗单页编辑积分。

## 11. 当前链路中的参数缺口

### 11.1 前端没有传递 visual_anchors

后端分镜接口已经返回：

```text
visual_anchors
```

后端创建绘本接口也已经支持：

```text
visual_anchors
```

但当前前端 `storyboardService.ts` 和 `storybookService.ts` 类型/请求体未完整传递该字段。

影响：

- `Storybook.visual_anchors` 可能为空。
- 正文生成时 `anchors_for_storyboard(...)` 无法解析当前页锚点。
- 调试台无法还原真实锚点输入。

### 11.2 has_character_reference_images 未在前端传递

后端分镜请求支持：

```text
has_character_reference_images
```

但当前前端生成分镜时没有传。

影响：

- 如果用户后续上传角色参考图，分镜阶段仍可能生成 `character` 锚点。
- 与“有角色参考图时不生成角色锚点”的规则不一致。

### 11.3 角色参考图未持久化

创建绘本后台任务收到：

```text
req.images
```

并作为 `character_reference_images` 传入每页生图。

但当前 `Storybook` 没有持久化保存这些角色参考图。

影响：

- 后续调试台无法复原首生成时的角色参考图。
- 单页重新生成默认也无法复用创建时的角色参考图。

### 11.4 首生成与单页重生的参考图语义不一致

首生成时：

```text
character_reference_images = 用户上传的角色参考图
```

单页重生时：

```text
character_reference_images = reference_page_ids 选中的正文页图片
```

虽然复用了同一个参数名，但语义不同。后续调试台和代码重构时建议拆分为：

```text
character_reference_images
selected_reference_page_images
previous_page_image
style_reference_images
```

## 12. 调试台需要采集的最小生成快照

为了准确调试生成效果，每次生成至少需要采集：

| 类别 | 字段 |
|---|---|
| 用户输入 | `instruction`、故事标题、故事正文 |
| 创建参数 | `cli_type`、`aspect_ratio`、`image_size` |
| 图片风格 | `image_style_id`、`image_style_version_id`、画风 prompt、画风参考图 |
| 角色参考图 | 创建时上传的角色参考图 |
| 锚点 | `Storybook.visual_anchors`、当前页 resolved anchors |
| 分镜 | `Page.storyboard` |
| 页面文本 | `Page.text` |
| 上下文 | 全部正文页文本 |
| 图片连续性 | 上一正文页图片 |
| 输出 | 当前页生成图片 URL、错误信息 |

这些快照应来自真实链路自动采集，不应依赖管理员手工复写。
