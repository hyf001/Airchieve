# 图片风格改造方案

## 1. 背景

当前系统中存在“模版”概念，但该概念更接近绘本内容或项目配置，无法准确表达 AI 图片生成中“参考某种视觉风格”的能力。

随着图片生成能力演进，系统需要支持通过一组风格样例图、风格描述和生成约束，稳定生成同一视觉风格下的图片。因此，建议将原有“图片模版”能力调整为“图片风格”。

## 2. 核心定义

图片风格是由管理员维护的 AI 绘图风格资产，用于指导模型生成具有一致视觉特征的图片。

图片风格主要描述：

- 色彩倾向
- 笔触质感
- 角色造型规律
- 构图方式
- 光影氛围
- 材质纹理
- 整体画面调性

图片风格不负责描述故事内容，也不绑定某一本绘本。

## 3. 设计目标

1. 将“模版”从绘本内容概念中拆出，升级为独立的“图片风格”资产。
2. 支持一个风格关联多张参考图，作为模型生成时的视觉样例。
3. 支持风格版本管理，保证已生成绘本后续页面生成时保持画风一致。
4. 仅管理员可创建、编辑、启用、停用风格。
5. 普通用户只负责选择风格，不参与风格维护。
6. 绘本生成图片时，可基于选定风格保持整体视觉一致性。
7. 支持管理员上传参考图后，由模型自动生成风格描述、生成提示词和负面提示词。
8. 不迁移历史风格或旧模版数据，新图片风格体系从零开始建设。

## 4. 用户角色

### 4.1 管理员

管理员负责维护平台风格库，包括：

- 创建图片风格
- 上传风格参考图
- 设置风格封面
- 基于参考图自动生成风格描述、生成提示词和负面提示词
- 编辑风格描述
- 编辑生成提示词
- 编辑负面提示词
- 发布新版本
- 启用或停用风格
- 调整风格排序
- 试生成验证效果

### 4.2 普通用户

普通用户只能使用风格，包括：

- 浏览可用风格
- 查看风格样例图
- 在创建或编辑绘本时选择风格
- 使用该风格生成绘本图片

普通用户不能创建、编辑或删除风格。

## 5. 核心对象

### 5.1 图片风格 ImageStyle

表示一个稳定的风格资产。

建议字段：

- 风格 ID
- 风格名称
- 风格封面图
- 风格标签
- 当前生效版本 ID
- 状态：启用 / 停用
- 排序权重
- 创建时间
- 更新时间

图片风格本身更像一个容器，真正用于生成的是某个具体版本。

### 5.2 图片风格版本 ImageStyleVersion

表示风格在某一时刻的固定配置。

建议字段：

- 版本 ID
- 风格 ID
- 版本号，例如 v1、v2、v3
- 画风摘要
- 风格描述
- 生成提示词
- 负面提示词
- 自动解析结果
- 封面图
- 状态：草稿 / 已发布
- 创建人
- 创建时间
- 发布时间

绘本应引用具体的风格版本，而不是只引用风格 ID。

### 5.3 图片风格参考图 ImageStyleReferenceImage

表示某个风格版本下的一张参考图。参考图需要独立建表，避免后续修改图片、删除图片、调整封面图或排序时必须整体覆盖版本配置 JSON。

建议字段：

- 参考图 ID
- 风格版本 ID
- 图片地址
- 是否封面图
- 排序
- 图片备注，可选
- 创建人
- 创建时间
- 更新时间

参考图与风格版本是一对多关系：

```text
ImageStyleVersion
  └── ImageStyleReferenceImage[]
```

## 6. 画风分类

首批画风不建议过多，应优先覆盖绘本中最常见、差异明显、用户容易理解的视觉类型。

画风分类应围绕“视觉表达方式”设计，不围绕主角类型设计。同一个画风可以适用于人类主角、动物主角、拟人化角色或其他角色。

例如：

```text
柔和水彩绘本风
- 小女孩主角
- 小狗主角
- 小兔子主角
- 小机器人主角
```

画风负责“怎么画”，角色设定负责“画谁”。

### 6.1 首批推荐画风

第一阶段建议先做 6 个系统画风：

1. 柔和水彩绘本风
2. 明亮卡通插画风
3. 极简幼儿涂鸦风
4. 蜡笔 / 彩铅手绘风
5. 国风儿童插画风
6. 梦幻奇幻绘本风

这 6 个画风覆盖：

- 通用温暖类：柔和水彩绘本风
- 低龄活泼类：明亮卡通插画风
- 极简低幼类：极简幼儿涂鸦风
- 手绘质感类：蜡笔 / 彩铅手绘风
- 中文内容特色：国风儿童插画风
- 想象力故事：梦幻奇幻绘本风

### 6.2 画风说明

#### 柔和水彩绘本风

适合睡前故事、自然、家庭、治愈类绘本。

视觉特点：

- 低饱和
- 水彩纸纹
- 柔和边缘
- 温暖光感
- 背景细节适中

#### 明亮卡通插画风

适合低龄儿童、搞笑故事、动物角色、轻松冒险。

视觉特点：

- 颜色鲜明
- 线条干净
- 角色圆润
- 表情夸张
- 画面识别度高

#### 极简幼儿涂鸦风

适合 2-5 岁低龄绘本、认知类绘本、简单情绪表达。

视觉特点：

- 元素少
- 背景简单
- 大色块
- 线条稚拙
- 主体非常明确

#### 蜡笔 / 彩铅手绘风

适合校园、家庭、日常、成长类绘本。

视觉特点：

- 颗粒感
- 手绘线条
- 色彩柔和但活泼
- 质感亲切
- 适合表现儿童日常

#### 国风儿童插画风

适合中国传统故事、节气、神话、成语故事。

视觉特点：

- 水墨或淡彩质感
- 东方构图
- 柔和线条
- 传统纹样或中式场景元素
- 整体克制、雅致

#### 梦幻奇幻绘本风

适合魔法、森林、星空、精灵、想象力故事。

视觉特点：

- 柔和发光元素
- 梦幻色彩
- 氛围感强
- 适合奇幻场景
- 画面想象力更强

### 6.3 可后续扩展画风

后续可以根据内容需求再增加：

- 黏土 / 软陶定格风
- 剪纸拼贴风
- 复古童书插画风
- 黑白线稿故事风

这些画风有明确特色，但首期不是必需能力，可以等基础画风稳定后再补充。

### 6.4 暂不建议的画风

第一阶段不建议做：

- 写实摄影风
- 强 3D 游戏风
- 赛博朋克风
- 暗黑恐怖风
- 复杂油画大师风
- 明确模仿某个在世艺术家的风格
- 过度二次元风

这些画风要么与儿童绘本场景不贴合，要么生成稳定性、审核风险或版权风险更高。

### 6.5 画风样例图建议

一个画风不应只上传同一种主体的样例图，否则模型可能把主体误认为画风的一部分。

推荐每个画风的样例图覆盖：

- 1 张人物主角图
- 1 张动物主角图
- 1 张场景图
- 1 张多角色互动图
- 1 张材质或细节图

这样模型更容易学习视觉风格，而不是复制某个主角。

## 7. 风格自动解析

管理员创建或编辑风格版本时，应支持上传一张或多张参考图，然后由多模态模型自动分析图片，生成该风格版本的基础配置。

自动解析的目标不是描述图片内容本身，而是抽象出可复用的视觉风格，供后续图片生成使用。

### 7.1 输入

管理员提供：

- 风格名称，可选
- 一张或多张风格参考图
- 单图备注，可选
- 管理员补充说明，可选

### 7.2 输出

系统自动生成：

- 风格名称建议
- 风格标签
- 风格描述
- 生成提示词
- 负面提示词
- 适用场景建议
- 主参考图建议

其中：

- 风格描述用于给用户和管理员理解该风格。
- 生成提示词用于拼接到图片生成请求中。
- 负面提示词用于约束模型避免生成不符合风格或风险较高的内容。

### 7.3 生成内容要求

风格描述应聚焦：

- 色彩倾向
- 笔触和线条
- 材质纹理
- 光影氛围
- 角色造型规律
- 背景复杂度
- 构图特点

生成提示词应表达：

- 需要学习参考图的整体视觉风格
- 需要保持绘本插画的一致性
- 需要结合当前页面内容生成新画面
- 不要复制参考图中的具体人物、构图、文字或标识

负面提示词应约束：

- 不要复刻参考图具体角色
- 不要生成水印、logo、乱码文字
- 不要出现风格冲突的摄影感、3D 渲染感或过强写实感，除非该风格本身需要
- 不要生成低质量、模糊、畸形、比例错误的画面

### 7.4 人工确认

自动解析结果必须允许管理员修改。

推荐流程：

1. 管理员上传参考图。
2. 点击“自动生成风格配置”。
3. 系统生成风格描述、提示词、负面提示词和标签。
4. 管理员检查并调整内容。
5. 管理员保存草稿或发布版本。

自动解析只是辅助能力，最终发布内容以管理员确认后的配置为准。

## 8. 版本管理设计

风格需要版本管理，原因是：

当管理员修改风格参考图、提示词或描述后，后续生成效果可能发生变化。如果没有版本管理，历史绘本无法知道当时使用的是哪一套风格配置。

因此推荐设计为：

```text
图片风格 ImageStyle
  └── 图片风格版本 ImageStyleVersion
        └── 参考图列表
```

### 8.1 版本规则

1. 新建风格时，默认创建 v1。
2. 已发布版本不可直接修改。
3. 修改已发布风格时，应创建新版本。
4. 管理员确认后，将新版本发布为当前生效版本。
5. 已生成绘本继续引用其生成时使用的旧版本。
6. 新创建绘本默认使用当前生效版本。

### 8.2 示例

```text
柔和水彩童话风
  - v1：旧参考图 + 旧提示词
  - v2：新增人物参考图 + 优化负面提示词

绘本 A 使用 v1
绘本 B 使用 v2
```

这样可以保证历史生成稳定，同时允许风格持续优化。

## 9. 多张参考图交互

### 9.1 管理员端

管理员编辑风格版本时，可以上传多张参考图。

建议交互：

1. 图片墙展示所有参考图。
2. 支持上传、删除、替换图片。
3. 支持拖拽排序。
4. 支持选择一张作为封面图。
5. 第一张或封面图作为主参考图。
6. 其他图片作为辅助参考图。
7. 支持一键自动解析，生成风格描述、生成提示词、负面提示词和标签。
8. 可选支持单图备注，例如“参考人物比例”“参考背景氛围”。

推荐限制：

```text
每个风格版本建议上传 3-8 张参考图
至少 1 张
最多 10 张
```

参考图类型建议覆盖：

- 人物图
- 场景图
- 材质细节图
- 光影氛围图
- 构图示例图

### 9.2 用户端

用户不需要选择具体参考图，只需要选择一个风格。

用户看到的是：

- 风格封面
- 风格名称
- 风格标签
- 简短描述
- 样例图预览

用户操作应保持简单：

```text
选择画风 -> 生成图片
```

不要让普通用户理解或管理多张参考图的权重。

## 10. 生成流程

用户生成绘本图片时，流程如下：

1. 用户选择绘本内容。
2. 用户选择图片风格。
3. 系统读取该风格的当前生效版本。
4. 系统获取该版本的参考图、风格描述、提示词和负面提示词。
5. 系统结合当前页面内容生成最终请求。
6. 调用图片生成模型。

当前使用的图片生成模型均具备多模态理解能力，因此图片生成阶段默认传入画风参考图，不设计“不支持图片参考”的纯文本替代链路。

用户选择画风后，系统不应只把画风名称传给模型，而是读取该画风当前版本配置，并根据不同生成阶段选择不同输入。

### 10.1 模型输入规则

画风版本配置可拆成两类输入：

- 分镜阶段输入：用于弱参考，帮助模型控制画面表达复杂度。
- 图片生成阶段输入：用于强约束，帮助模型生成一致画风的图片。

### 10.2 分镜阶段传给模型的内容

分镜阶段只传轻量画风信息。

建议传入：

- 画风名称
- 画风摘要
- 分镜复杂度建议，可选

示例：

```text
画风名称：柔和水彩绘本风
画风摘要：整体温暖、低饱和、儿童绘本式画面，适合柔和自然的场景表达。
分镜复杂度建议：中等复杂度，保留主体动作和环境氛围，不需要复杂摄影光影。
```

分镜阶段不要传：

- 风格参考图
- 完整风格描述
- 生成提示词
- 负面提示词

### 10.3 图片生成阶段传给模型的内容

图片生成阶段使用完整画风配置。

建议传入：

- 页面内容
- 角色设定
- 分镜描述
- 风格描述
- 生成提示词
- 负面提示词
- 参考图列表

示例：

```text
页面内容：小狗在雨后草地上发现一只蜗牛。
角色设定：一只黄色小狗，活泼，戴红色围巾。
分镜描述：小狗蹲在草地上看着蜗牛，画面重点是小狗好奇的表情和蜗牛。
风格描述：柔和的儿童绘本插画风格，低饱和暖色调，水彩纸纹明显，线条轻柔。
生成提示词：参考该画风的整体视觉特征，包括色彩、笔触、材质、角色造型规律、光影氛围和背景复杂度。不要复制参考图中的具体人物、构图或场景。
负面提示词：不要生成写实摄影风、3D 游戏风、水印、logo、乱码文字、畸形手脚、过度阴暗恐怖画面。
参考图列表：当前风格版本下的主参考图和辅助参考图。
```

图片生成阶段应传入参考图列表，由模型结合参考图、风格描述、生成提示词和负面提示词共同生成图片。

## 11. 现有流程改造

新图片风格体系需要接入现有绘本创建、图片生成和重新生成流程。改造重点是：现有流程不再依赖旧模版概念，而是在需要生成图片的地方统一读取图片风格版本。

### 11.1 绘本创建流程

现有绘本创建流程中，应增加“选择画风”步骤。

改造后流程：

1. 用户输入绘本主题、故事内容或其他创作信息。
2. 用户选择一个已启用的图片风格。
3. 系统保存绘本与图片风格的关系。
4. 系统生成绘本页面内容。
5. 系统生成页面图片时，使用该图片风格的当前生效版本。

绘本建议记录：

- 风格 ID
- 默认风格版本 ID

其中，默认风格版本 ID 用于保证同一本绘本后续继续生成时，默认沿用创建时选定的风格版本，避免管理员发布新版本后影响已创建绘本的整体一致性。

绘本生成后，画风应被锁定，不支持重新更换画风。

### 11.2 页面图片生成流程

现有页面图片生成逻辑需要从“只根据页面内容生成图片”调整为“页面内容 + 风格版本配置”共同生成。

生成前需要读取：

- 页面内容描述
- 绘本当前使用的风格 ID
- 绘本当前使用的风格版本 ID
- 风格版本下的参考图
- 风格描述
- 生成提示词
- 负面提示词

最终请求应由三部分组成：

```text
页面内容：当前页面需要画什么。
风格要求：该风格版本的描述和生成提示词。
参考图片：该风格版本下的主参考图和辅助参考图。
```

图片生成时应传入风格版本下的参考图，并结合风格描述、生成提示词和负面提示词共同生成。

### 11.3 分镜生成流程

画风会影响分镜的表达倾向，但不应改变故事内容和页面事件。

分镜生成主要负责“这一页画什么”，包括：

- 页面事件
- 角色
- 动作
- 场景
- 情绪
- 画面重点

画风主要负责“这一页怎么画”，包括：

- 色彩
- 笔触
- 质感
- 角色造型规律
- 光影氛围
- 背景复杂度

推荐规则：

1. 分镜生成可以读取画风摘要，作为弱参考。
2. 分镜生成不传风格参考图。
3. 分镜生成不使用完整生成提示词和负面提示词。
4. 分镜生成不应因为画风改变故事内容、角色关系或页面事件。
5. 图片生成阶段再完整使用风格描述、生成提示词、负面提示词和参考图。

分镜阶段可使用的画风摘要示例：

```text
当前绘本使用“柔和水彩童话风”，整体适合温暖、低饱和、儿童绘本式画面。
```

分镜阶段不应使用：

- 风格参考图
- 完整生成提示词
- 负面提示词
- 参考图中的具体人物、构图或场景内容

### 11.4 低复杂度画风下的分镜适配

当画风为极简、低龄、幼稚、涂鸦等低复杂度风格时，分镜生成应弱化复杂镜头和光影描述，改为描述主体大小、画面重点、背景简化程度和情绪氛围。

这些描述仍然有用，但要从“摄影语言”转换为“画面意图”。

保留的通用分镜信息：

- 页面事件
- 角色和动作
- 场景
- 情绪
- 画面重点

需要弱化的分镜信息：

- 复杂镜头距离
- 专业摄影角度
- 复杂光影
- 景深虚化
- 过细的背景层次

推荐写法：

```text
画面简单，角色占画面主要位置，背景只保留少量树和天空，整体明亮温暖。
```

不推荐写法：

```text
低机位近景，夕阳逆光，角色边缘有强烈轮廓光，背景有明显景深虚化。
```

也就是说：

- “近景”可以转成“主体更大、表情更清楚”。
- “光影”可以转成“画面明亮、温暖、开心”。
- “构图”可以转成“主体居中、背景简单、留白较多”。

### 11.5 页面重新生成流程

页面重新生成时，默认应沿用该绘本当前记录的风格版本。

推荐规则：

1. 页面重新生成时，不自动切换到图片风格的最新版本。
2. 页面重新生成不允许更换图片风格。
3. 页面重新生成不允许切换到图片风格的最新版本。

这样可以避免同一本绘本前后页面画风不一致。

### 11.6 画风锁定规则

绘本生成后不支持重新更换画风。

推荐规则：

1. 画风只能在绘本创建或生成前选择。
2. 绘本生成后，风格 ID 和默认风格版本 ID 不允许被普通用户修改。
3. 已生成绘本的页面重新生成必须沿用绘本锁定的风格版本。
4. 管理员发布新的风格版本，不影响已生成绘本。
5. 如果用户希望使用其他画风，应创建一本新的绘本。

绘本锁定字段：

- 风格 ID
- 默认风格版本 ID

### 11.7 接口与参数改造

现有生成接口应支持传入或推导图片风格。

推荐规则：

1. 创建绘本接口支持传入 `image_style_id`。
2. 图片生成接口优先使用绘本记录的 `image_style_version_id`。
3. 管理员试生成接口允许直接指定 `image_style_version_id`。
4. 如果缺少风格信息，应返回明确错误，不再隐式使用旧模版。
5. 旧模版相关参数不再进入新生成链路。

### 11.8 前端展示改造

用户侧需要在以下位置展示或使用图片风格：

- 创建绘本页：选择画风。
- 绘本详情页：展示当前画风。
- 页面重新生成入口：默认展示当前使用的画风。

用户侧不提供已生成绘本的更换画风入口。

管理员侧需要新增图片风格管理入口，旧模版管理入口不再作为新图片生成的配置入口。

## 18. 核心方法改造说明

### 18.1 画风可用性校验

新增公共方法：

```text
validate_style_available(style_id) -> ImageStyleVersion
```

职责：

1. 校验画风存在。
2. 校验画风已启用。
3. 校验存在当前已发布版本。
4. 校验版本具备画风摘要、风格描述、生成提示词、负面提示词和参考图。
5. 返回当前版本配置。

使用位置：

- 生成分镜前
- 创建绘本前
- 管理员试生成前

### 18.2 分镜生成方法

现有方法：

```text
create_storyboard_only(story_content, page_count, cli_type)
GeminiCli.create_storyboard_from_story(story_content, page_count)
```

改造后：

```text
create_storyboard_only(story_content, page_count, cli_type, image_style_id)
GeminiCli.create_storyboard_from_story(story_content, page_count, style_summary=None)
```

处理逻辑：

1. 如果传入 `image_style_id`，读取当前版本。
2. 只取 `style.name`、`style_summary` 和分镜复杂度建议。
3. 将摘要拼入分镜 prompt。
4. 不传参考图、不传生成提示词、不传负面提示词。

异常：

- 画风不可用：返回 `400`
- 分镜解析失败：沿用现有 `ValueError`

### 18.3 绘本创建方法

现有方法：

```text
create_storybook_from_story_async(..., template_id)
run_create_storybook_from_story_background(..., template, images)
```

改造后：

```text
create_storybook_from_story_async(..., image_style_id)
run_create_storybook_from_story_background(..., image_style_version_id, images)
```

处理逻辑：

1. 创建前校验 `image_style_id`。
2. 获取当前版本 ID。
3. 创建 `Storybook` 时写入 `image_style_id` 和 `image_style_version_id`。
4. 后台任务只使用绘本锁定的 `image_style_version_id`。
5. 不再查询 `Template`。

### 18.4 页面图片生成方法

现有方法：

```text
GeminiCli.generate_page(..., reference_images, previous_page_image, template, ...)
```

改造后：

```text
GeminiCli.generate_page(
  ...,
  character_reference_images=None,
  previous_page_image=None,
  image_style_version=None,
  ...
)
```

处理逻辑：

1. 页面内容和分镜仍然作为主体生成依据。
2. 角色参考图继续用于角色一致性。
3. 上一页图片继续用于同一本绘本内部连续性。
4. 画风参考图只用于学习视觉风格。
5. prompt 中明确“不要复制参考图中的具体人物、文字、构图或场景”。

### 18.5 封面生成方法

现有封面生成依赖内页图作为参考。

建议改造：

```text
generate_cover(..., reference_images, image_style_version=None)
```

处理逻辑：

1. 继续优先使用内页图保持角色和画面一致。
2. 同时加入画风描述和生成提示词。
3. 若内页图不足，则使用画风参考图兜底。

### 18.6 自动解析方法

新增：

```text
analyze_style_from_images(reference_images, name=None, notes=None)
```

处理逻辑：

1. 校验图片数量、格式和大小。
2. 调用多模态模型分析图片。
3. 要求模型输出结构化 JSON。
4. 服务层校验字段完整性。
5. 返回给管理员编辑，不直接入库。

输出字段：

- 风格名称建议
- 标签
- 画风摘要
- 风格描述
- 生成提示词
- 负面提示词
- 适用场景建议

## 19. 异常场景处理

### 19.1 画风不存在

场景：

- 用户创建绘本时传入不存在的 `image_style_id`
- 分镜生成时传入不存在的 `image_style_id`

处理：

- 返回 `404`
- 文案：`画风不存在`

### 19.2 画风已停用

场景：

- 普通用户选择了已停用画风
- 前端缓存了旧画风列表

处理：

- 创建绘本和分镜生成返回 `400`
- 文案：`该画风已停用，请重新选择画风`

说明：

- 已生成绘本不受影响，仍然使用锁定的风格版本。

### 19.3 画风没有已发布版本

场景：

- 管理员创建了画风但未发布版本

处理：

- 用户侧列表不展示该画风。
- 如果接口被直接调用，返回 `400`
- 文案：`该画风暂不可用`

### 19.4 画风版本配置不完整

场景：

- 缺少画风摘要
- 缺少风格描述
- 缺少生成提示词
- 缺少负面提示词
- 缺少参考图

处理：

- 发布版本时返回 `422`
- 创建绘本时如遇脏数据返回 `400`
- 文案：`画风配置不完整，请联系管理员`

### 19.5 参考图无法读取

场景：

- 图片 URL 失效
- 图片 base64 格式错误
- 图片过大
- 图片类型不是图片

处理：

- 自动解析接口返回 `422`
- 生图阶段如果部分参考图失效，应跳过失效图片并记录日志。
- 如果全部参考图失效，应中止本次生成并返回错误，避免画风失控。
- 文案：`画风参考图不可用，请联系管理员检查画风配置`

### 19.6 自动解析失败

场景：

- 模型未返回 JSON
- 字段缺失
- 模型安全拒绝
- 模型超时

处理：

- JSON 解析失败：`500` 或 `422`
- 安全拒绝：`400`
- 超时：`504`
- 管理端提示管理员重新上传图片或手动填写配置

### 19.7 生成图片失败

场景：

- 大模型拒绝
- 超时
- 未返回图片
- 参考图导致生成失败

处理：

- 沿用现有 `LLMError` 机制。
- 当前页面状态置为 `error`。
- 绘本状态根据页面状态同步。
- 用户可页面重新生成，但仍使用绘本锁定的画风版本。

### 19.8 请求仍传 template_id

场景：

- 老前端或小程序仍传 `template_id`

处理：

- 新接口忽略 `template_id` 或返回 `400`。
- 推荐返回 `400`，文案：`当前生成流程已改为使用画风，请选择画风`

### 19.9 已生成绘本尝试更换画风

场景：

- 前端误传新的 `image_style_id`
- 用户尝试通过接口修改画风

处理：

- 返回 `400`
- 文案：`绘本生成后不支持更换画风`

### 19.10 画风版本被管理员更新

场景：

- 管理员发布了 v2，但某绘本绑定的是 v1

处理：

- 已生成绘本继续使用 v1。
- 新创建绘本使用当前生效版本 v2。
- 页面重新生成不自动切换版本。

## 22. 最终结论

图片风格应作为独立的 AI 图片生成资产，由管理员统一维护。

一个图片风格可以包含多个版本，每个版本包含多张参考图、风格描述、提示词和负面提示词。

管理员上传参考图后，系统应支持自动解析图片风格，并生成可编辑的风格描述、生成提示词、负面提示词和标签。

普通用户只需要选择风格，系统在生成图片时自动使用当前生效版本，从而保证绘本图片的风格一致性。

本次改造不迁移历史风格和旧模版数据，新图片风格体系从零开始建设。
