# 图片风格三期实施方案：风格图片库与管理员风格管理

## 1. 目标

三期补齐两类管理员能力：

- 风格图片库：上传、浏览、编辑、管理可复用的风格图片资产。
- 风格管理：创建风格版本时，从图片库选择参考图并发布版本。

本阶段完成后：

- 管理员可上传图片到 OSS，并沉淀为风格图片资产。
- 管理员可为图片资产设置名称、描述、标签和分类信息。
- 管理员可查看全部风格，包括启用、停用、未发布风格。
- 管理员可创建、编辑、启用、停用风格。
- 管理员可创建和编辑草稿版本。
- 管理员可从图片库选择图片作为草稿版本参考图。
- 管理员可发布草稿版本，使其成为当前生效版本。

不做：

- 自动解析画风。
- 管理员试生成。
- 小程序管理页。
- 拖拽排序、批量操作。
- 复制版本为新草稿。

## 2. 核心规则

### 2.1 图片库资产可复用

风格图片先进入图片库，再被风格版本引用。

这样可以避免同一张图片在多个版本中重复上传，也能减少“上传后未保存草稿”的 OSS 孤儿文件问题。

### 2.2 已发布版本不可变

绘本生成会锁定 `image_style_version_id`。为了保证后续页面重新生成画风一致，已发布版本必须只读。

规则：

- 已发布版本不能编辑画风配置。
- 已发布版本不能新增、删除、修改参考图引用关系。
- 图片库资产的名称、描述、标签可以调整，但不要覆盖原 OSS 图片文件。
- 修改画风必须创建新版本。
- 发布新版本只更新 `ImageStyle.current_version_id`，不影响已生成绘本。

### 2.3 草稿版本可编辑

草稿版本允许：

- 编辑 `style_summary`、`style_description`、`generation_prompt`、`negative_prompt`。
- 从图片库添加或移除参考图。
- 设置封面和排序。
- 发布前做完整性校验。

### 2.4 封面规则

- 管理页列表封面使用 `ImageStyle.cover_image`。
- 发布版本时，将版本封面参考图 URL 同步到 `ImageStyle.cover_image`。
- 如果版本没有显式封面，发布时将 `sort_order` 最小的参考图设为封面。

## 3. 风格描述与标签规范

风格描述和标签需要预先设计，避免管理员自由填写导致资产难检索、版本难复用。

### 3.1 风格版本描述字段

风格版本仍保存四个生成相关字段：

| 字段 | 用途 |
|---|---|
| `style_summary` | 短摘要，分镜阶段使用 |
| `style_description` | 完整视觉描述，图片生成阶段使用 |
| `generation_prompt` | 正向画风提示词 |
| `negative_prompt` | 负面提示词 |

建议 `style_description` 按固定结构填写：

```text
色彩：...
线条：...
材质/笔触：...
角色造型：...
构图：...
光影：...
适合题材：...
避免事项：...
```

### 3.2 图片资产标签

图片库标签用于检索和筛选。标签应优先从预设枚举中选择，允许少量补充自定义标签。

首版预设枚举可以先放在前端常量和后端 schema 校验中，不必单独建标签表。后续如果需要运营维护标签，再抽成独立配置表。

建议首批标签维度：

| 维度 | 示例 |
|---|---|
| 风格类型 | 水彩、卡通、涂鸦、彩铅、国风、奇幻 |
| 色彩 | 低饱和、明亮、暖色、冷色、柔和、高对比 |
| 笔触材质 | 纸纹、蜡笔、颗粒感、水墨、平涂、柔边 |
| 场景 | 森林、室内、校园、夜晚、节日、传统建筑 |
| 主体 | 儿童、动物、植物、家庭、幻想角色 |
| 构图 | 大主体、留白、全景、近景、中心构图 |
| 年龄段 | 低幼、儿童、亲子 |

### 3.3 风格基础标签

`ImageStyle.tags` 用于用户侧理解风格，应比图片资产标签更少、更概括。

建议每个风格 2-5 个标签，例如：

```text
水彩、温暖、睡前故事、柔和
```

## 4. 数据模型

### 4.1 新增风格图片资产

新增表：

```text
ImageStyleAsset
```

建议字段：

```text
id
url
object_key
name
description
tags
style_type
color_tags
texture_tags
scene_tags
subject_tags
composition_tags
age_group_tags
content_type
file_size
width
height
is_active
creator
modifier
created_at
updated_at
```

说明：

- `url` 保存 OSS URL。
- `object_key` 保存 OSS key，方便代理访问和后续清理。
- 多维标签可先用 JSON 数组存储。
- `is_active=false` 表示资产下架，不在选择器中展示。
- 上传接口、编辑接口和模型字段需要保持一致。首版如果使用多维标签，请同时支持 `style_type`、`color_tags`、`texture_tags` 等字段，不要只提交 `tags`。

### 4.2 版本参考图改为引用资产

调整：

```text
ImageStyleReferenceImage
```

建议字段：

```text
id
image_style_version_id
asset_id
url_snapshot
is_cover
sort_order
note
creator
created_at
updated_at
```

说明：

- `asset_id` 指向 `ImageStyleAsset`。
- `url_snapshot` 保存发布时使用的图片 URL 快照，避免后续资产元信息变化影响版本。
- 响应给前端时建议额外返回解析后的 `url` 字段。草稿版本取资产 `ImageStyleAsset.url`，已发布版本优先取 `url_snapshot`。
- 已发布版本的引用关系不可变。

## 6. 后端接口

### 6.1 风格图片库

新增上传接口：

```http
POST /api/v1/image-style-assets/upload
Content-Type: multipart/form-data
```

权限：管理员。

字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `file` | `UploadFile` | 是 | 图片文件 |
| `name` | `string` | 否 | 图片名称 |
| `description` | `string` | 否 | 图片描述 |
| `tags` | `string[]` | 否 | 通用标签 |
| `style_type` | `string` | 否 | 风格类型 |
| `color_tags` | `string[]` | 否 | 色彩标签 |
| `texture_tags` | `string[]` | 否 | 笔触材质标签 |
| `scene_tags` | `string[]` | 否 | 场景标签 |
| `subject_tags` | `string[]` | 否 | 主体标签 |
| `composition_tags` | `string[]` | 否 | 构图标签 |
| `age_group_tags` | `string[]` | 否 | 年龄段标签 |

校验：

- 只允许 `image/png`、`image/jpeg`、`image/webp`。
- 单文件最大 10MB。

OSS key：

```text
image-style-assets/{yyyy}/{mm}/{uuid}.{ext}
```

上传成功后：

- 上传文件到 OSS。
- 创建 `ImageStyleAsset` 记录。
- 返回资产详情。

图片库接口：

```http
GET    /api/v1/image-style-assets
GET    /api/v1/image-style-assets/{asset_id}
PUT    /api/v1/image-style-assets/{asset_id}
DELETE /api/v1/image-style-assets/{asset_id}
```

规则：

- 列表支持按标签、风格类型、启用状态分页筛选。
- `DELETE` 只允许删除未被任何版本引用的资产。
- 已被引用的资产只能下架，即 `is_active=false`。

### 6.2 管理员风格列表

新增：

```http
GET /api/v1/image-styles/admin
```

要求：

- 返回启用、停用、未发布风格。
- 不强制 join 当前已发布版本。
- `current_version_id`、`current_version_no` 允许为空。
- 普通用户接口 `GET /api/v1/image-styles` 保持只返回可用风格。

### 6.3 更新草稿版本

新增：

```http
PUT /api/v1/image-styles/{style_id}/versions/{version_id}
```

规则：

- 只允许更新 `draft` 版本。
- 已发布版本返回 `400`：`已发布版本不可编辑，请创建新版本`。

请求体支持部分更新：

```json
{
  "style_summary": "...",
  "style_description": "...",
  "generation_prompt": "...",
  "negative_prompt": "..."
}
```

### 6.4 删除草稿版本

建议新增：

```http
DELETE /api/v1/image-styles/{style_id}/versions/{version_id}
```

规则：

- 只允许删除 `draft` 版本。
- 已发布版本不允许删除。
- 删除草稿版本时同时删除该版本的参考图引用关系。
- 不删除图片库资产本身。

### 6.5 版本参考图引用

新增或调整参考图接口：

```http
POST   /api/v1/image-styles/{style_id}/versions/{version_id}/reference-images
PUT    /api/v1/image-styles/{style_id}/versions/{version_id}/reference-images/{image_id}
DELETE /api/v1/image-styles/{style_id}/versions/{version_id}/reference-images/{image_id}
```

创建请求改为引用图片库资产：

```json
{
  "asset_id": 12,
  "is_cover": false,
  "sort_order": 0,
  "note": "色彩参考"
}
```

规则：

- 只允许操作草稿版本。
- 已发布版本返回 `400`：`已发布版本的参考图不可修改，请创建新版本`。
- 新增参考图时，如果请求未传 `sort_order`，后端自动取当前版本下最大 `sort_order + 1`。
- 删除草稿封面图后，若仍有剩余参考图，自动将 `sort_order` 最小的一张设为封面。

### 6.6 生成链路读取参考图

生成服务不应关心参考图来自 `asset_id` 还是 `url_snapshot`，建议在 `image_style_service.py` 中提供统一方法：

```python
async def get_reference_image_urls(version_id: int) -> list[str]:
    ...
```

规则：

- 已发布版本：优先读取 `ImageStyleReferenceImage.url_snapshot`。
- 草稿版本：通过 `asset_id` 关联读取 `ImageStyleAsset.url`，用于后续管理员试生成。
- 如果 `url_snapshot` 为空，可回退到资产 URL。

Gemini / Doubao 生成服务只接收 URL 列表，不直接读取 ORM 字段，避免调用方分散判断版本状态。

### 6.7 发布版本

沿用：

```http
POST /api/v1/image-styles/{style_id}/versions/{version_id}/publish
```

发布校验：

- `style_summary` 非空。
- `style_description` 非空。
- `generation_prompt` 非空。
- `negative_prompt` 非空。
- 至少 1 张参考图。

发布成功：

- `version.status = "published"`。
- `version.published_at = now`。
- `ImageStyle.current_version_id = version.id`。
- 将参考图资产 URL 写入 `url_snapshot`。
- 同步版本封面参考图 URL 到 `ImageStyle.cover_image`。

- 点击“选择参考图”打开图片库选择器。

- 点击已发布行的“查看”按钮进入只读详情视图。

```typescript
listAdminImageStyles(params): Promise<AdminImageStyleListItem[]>
```

`ReferenceImage` 响应建议包含解析后的 `url`：

```typescript
interface ReferenceImage {
  id: number;
  image_style_version_id: number;
  asset_id: number;
  url: string;
  url_snapshot: string | null;
  is_cover: boolean;
```

前端展示缩略图统一使用 `referenceImage.url`，不判断版本状态。

- 使用 `FormData`。
- 请求头只加鉴权头，不要手动设置 `Content-Type`。
- 图片展示继续使用 `toApiUrl(url)`。

## 10. 验收标准

后端：

1. 管理员可上传图片并创建图片资产。
2. 图片资产可列表、筛选、编辑。
3. 未被引用的图片资产可删除。
4. 已被引用的图片资产不能物理删除，只能下架。
5. 管理员列表返回全部风格，包括停用和未发布。
6. 草稿版本配置可更新。
7. 草稿版本可删除。
8. 已发布版本配置不可更新。
9. 已发布版本参考图引用不可修改。
10. 草稿版本可从图片库添加参考图。
11. 服务层可统一返回版本参考图 URL 列表。
12. 发布版本会校验完整性。
13. 发布后更新 `current_version_id` 和 `cover_image`。

前端：

1. 管理员可进入风格图片库。
2. 管理员可上传、编辑、筛选图片资产。
3. 管理员可进入图片风格管理页。
4. 管理员可创建和编辑风格。
5. 管理员可启用和停用风格。
6. 管理员可创建、编辑和删除草稿版本。
7. 管理员可从图片库选择参考图。
8. 管理员可删除参考图引用、设置封面。
9. 草稿和已发布版本的参考图缩略图都能正常展示。
10. 已发布版本只读。
11. 管理员可发布完整草稿版本。
12. 发布后列表封面和当前版本刷新正确。

回归：

1. 首页导航正常。
2. 用户管理页面正常。
3. 模版管理页面正常。
4. 用户侧绘本创建流程正常。
5. 页面重新生成仍使用绘本锁定的风格版本。
